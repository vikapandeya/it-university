File System Concepts

  • File — named collection of data stored on disk
  • Folder/Directory — container for organizing files
  • Path — location of a file, e.g. C:\Users\John\Documents\report.pdf
  • Extension — type indicator (.pdf, .docx, .jpg, .py, .exe)
  • Hidden files — prefixed with dot on Linux (.bashrc), toggled on Windows
  • Permissions — who can read/write/execute a file (owner, group, others)

Email Best Practices

  • Subject line — specific and actionable: "Q3 Report Review — Feedback by Friday"
  • CC vs BCC — CC is visible to all recipients; BCC is hidden (use for privacy)
  • Reply-All caution — only Reply All when the whole thread needs your response
  • Attachments — mention in body; use links for large files (>10MB)
  • Phishing — never click links or download files from unexpected emails
  • Professional tone — greeting, body, signature; proofread before sending

Internet Safety

  • Use strong unique passwords + a password manager (Bitwarden, 1Password)
  • Enable 2FA (Two-Factor Authentication) on all important accounts
  • Check for HTTPS (padlock icon) before entering passwords
  • Keep browser and OS updated — patches fix security vulnerabilities
  • Use a VPN on public WiFi (airports, cafes, hotels)
  • Never share personal info in reply to unsolicited messages

PowerPoint — Key Skills

  • Slide Master (View → Slide Master) for consistent theme across all slides
  • SmartArt for org charts and process diagrams
  • Animations (per element) and Transitions (per slide)
  • Presenter View — see notes + timer + next slide while audience sees only slides
  • Export as PDF or video (.mp4): File → Export
  • Embed videos, audio, and clickable hyperlinks

Presentation Design Rules

  • 6×6 Rule — max 6 bullet points per slide, 6 words per bullet
  • Fonts — max 2 fonts; 28pt minimum for body text
  • Contrast — dark text on light background or vice versa, never similar shades
  • Images over text — a relevant image communicates faster than bullets
  • Timing — 1–2 minutes per slide; use Rehearse Timings to practice
  • White space — don't crowd slides; give content room to breathe

🔧 PC Maintenance & Troubleshooting

Keep Your PC Running Fast

Regular Maintenance Checklist

  • ✅ Windows Update — monthly; critical patches fix security holes
  • ✅ Disk Cleanup — delete temp files (Win+R → cleanmgr)
  • ✅ Startup programs — Task Manager → Startup; disable unused apps
  • ✅ Antivirus scan — weekly full scan (Windows Defender is sufficient)
  • ✅ Browser cleanup — clear cache/cookies monthly
  • ✅ Backup — 3-2-1 rule: 3 copies, 2 different media, 1 offsite/cloud
  • ✅ Physical cleaning — compressed air for vents quarterly

Common Problems & Solutions

ProblemCommon CauseSolution
Computer slowFull RAM/disk, startup bloatClose unused apps, disable startup, add RAM
No internetDriver, router, DNS issueRestart router, run network troubleshooter
Blue screen (BSOD)Driver crash, hardware faultNote error code, update drivers, check RAM
OverheatingDust, faulty fan, bad pasteClean vents, reapply thermal paste
App won't openCorrupt install, missing DLLRepair install via Add/Remove Programs

📖 Computer Glossary A–Z

Essential IT Terms
ABCD EFHI LMNO PRST UVW
A
API (Application Programming Interface) — A set of rules for how software applications communicate with each other. Like a menu in a restaurant — lists what you can order (request) and what you'll get back.
Algorithm — A step-by-step procedure for solving a problem or completing a task. Like a recipe for code.
B
Bandwidth — The maximum data transfer rate of a network connection. Measured in Mbps or Gbps.
Bit / Byte — Bit = smallest unit (0 or 1). Byte = 8 bits. 1 KB = 1024 bytes, 1 MB = 1024 KB.
Bluetooth — Short-range wireless technology (typically 10m) for connecting devices (headphones, keyboards, mice).
C
Cache — Temporary storage for frequently accessed data. Web browsers cache pages; CPUs cache instructions for speed.
Cloud Computing — Using remote servers (via the internet) to store, manage, and process data instead of local hardware. Examples: AWS, Azure, Google Cloud.
CPU (Central Processing Unit) — The primary chip that executes program instructions. The "brain" of the computer.
Cybersecurity — Protecting systems, networks, and programs from digital attacks, unauthorized access, and data theft.
D
Database — Organized collection of structured data. Types: Relational (SQL tables), NoSQL (documents, key-value).
DNS (Domain Name System) — The internet's phone book — translates human-readable domain names (google.com) to IP addresses (142.250.80.46).
E
Encryption — Converting data into a coded form that can only be read by authorized parties with a key. Used in HTTPS, passwords.
Ethernet — Wired networking technology using cables (Cat5e, Cat6) for high-speed, reliable connections.
F
Firewall — Network security system that monitors and controls incoming/outgoing traffic based on security rules.
Framework — Pre-built code structure that provides a foundation for application development. Examples: React (frontend), Django (backend).
H
Hardware — Physical components of a computer: CPU, RAM, SSD, motherboard, monitor, keyboard, mouse.
HTTP/HTTPS — HyperText Transfer Protocol — the foundation of web communication. HTTPS adds TLS encryption (padlock in browser).
I
IP Address — Unique numerical label for every device on a network. IPv4: 192.168.1.1 (4 octets). IPv6: longer hex format for more addresses.
IoT (Internet of Things) — Network of physical devices embedded with sensors/software that connect and exchange data over the internet.
L
LAN (Local Area Network) — Network connecting computers in a limited area (office, home). Uses switches and routers.
M
Malware — Malicious software: viruses, trojans, ransomware, spyware, adware. Spread via email attachments, downloads, websites.
Memory (RAM) — Random Access Memory — temporary, volatile storage where the CPU keeps data it's currently using. More RAM = smoother multitasking.
N
Network — Group of interconnected devices that communicate and share resources (files, printers, internet).
O
Open Source — Software with publicly available source code that anyone can view, modify, and distribute. Examples: Linux, Firefox, Python.
OS (Operating System) — Software that manages hardware and provides services for applications. Examples: Windows, Linux, macOS, Android.
P
Phishing — Social engineering attack via fraudulent emails/messages designed to steal credentials, credit cards, or install malware.
Protocol — Set of rules governing how data is transmitted. Examples: HTTP, TCP/IP, FTP, SMTP, SSH.
R
Router — Networking device that forwards packets between networks. Connects your LAN to the internet via the ISP.
S
SaaS (Software as a Service) — Cloud-delivered software accessed via browser. Examples: Gmail, Salesforce, Microsoft 365, Slack.
Software — Programs and operating information used by a computer. Opposite of hardware.
SSL/TLS — Protocols that encrypt data in transit. TLS 1.3 is current. Creates the HTTPS padlock in browsers.
T
TCP/IP — Transmission Control Protocol/Internet Protocol — the fundamental communication protocol suite of the internet.
U
UI/UX — User Interface (visual design) / User Experience (overall feel and usability). Good UX makes software intuitive and satisfying.
USB — Universal Serial Bus — standard connector for peripherals (drives, keyboards, charging). USB-A, USB-C, USB 3.2 are common types.
V
Virtualization — Running multiple virtual machines on one physical host using software (VMware, VirtualBox, Hyper-V).
VPN (Virtual Private Network) — Encrypts your internet traffic and routes it through a remote server, masking your IP and location.
W
WiFi — Wireless networking using radio waves (802.11 standard). WiFi 6 (802.11ax) offers up to 9.6 Gbps theoretical speed.
WAN (Wide Area Network) — Network spanning large geographic areas. The internet is the largest WAN.
