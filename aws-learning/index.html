🔑 IAM — Identity and Access Management

Policy Evaluation Logic

IAM evaluates policies in this priority order when determining Allow/Deny:

  1. Explicit Deny in any policy — DENY (cannot be overridden)
  2. SCPs (Service Control Policies) — Organization-level. Must Allow.
  3. Resource-based policy with explicit Allow (cross-account case)
  4. IAM permissions boundary — defines max permissions
  5. Session policies — inline policy during AssumeRole
  6. Identity-based policies (user/group/role policies)
  7. Default — DENY

⚡ DynamoDB

Data Modeling, RCU/WCU, Single-Table Design

RCU / WCU Calculations

# Read Capacity Unit (RCU):
# 1 RCU = 1 strongly consistent read/sec for item up to 4KB
# 1 RCU = 2 eventually consistent reads/sec for item up to 4KB
# Example: 100 strongly consistent reads/sec, avg item 6KB
# RCUs = 100 * ceil(6/4) = 100 * 2 = 200 RCU

# Write Capacity Unit (WCU):
# 1 WCU = 1 write/sec for item up to 1KB

Single-Table Design Pattern

# Store multiple entity types with overloaded PK/SK
# PK              | SK                | attributes
# USER#u123       | PROFILE           | name, email
# USER#u123       | ORDER#o456        | total, status
# PRODUCT#p001    | METADATA          | name, price
# PRODUCT#p001    | REVIEW#r321       | rating, text

import boto3
from boto3.dynamodb.conditions import Key, Attr

dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('AppTable')

# Query all orders for a user (newest first)
response = table.query(
    KeyConditionExpression=Key('PK').eq('USER#u123') &
                          Key('SK').begins_with('ORDER#'),
    ScanIndexForward=False,
    Limit=10
)

# Conditional update (optimistic locking)
table.update_item(
    Key={'PK': 'PRODUCT#p001', 'SK': 'METADATA'},
    UpdateExpression='SET price = :p, version = version + :inc',
    ConditionExpression=Attr('version').eq(current_version),
    ExpressionAttributeValues={':p': 29.99, ':inc': 1}
)

# Transactional write (atomic multi-item)
dynamodb.meta.client.transact_write(
    TransactItems=[
        {'Put': {'TableName': 'AppTable', 'Item': order_item}},
        {'Update': {
            'TableName': 'AppTable',
            'Key': {'PK': 'USER#u123', 'SK': 'STATS'},
            'UpdateExpression': 'SET order_count = order_count + :inc',
            'ExpressionAttributeValues': {':inc': 1}
        }}
    ]
)

Caching Strategies

# Cache-Aside (Lazy Loading) — most common
import redis, json, boto3

r = redis.Redis(host='cluster.xxx.cache.amazonaws.com', port=6379)

def get_user(user_id):
    key = f"user:{user_id}"
    cached = r.get(key)
    if cached:
        return json.loads(cached)
    user = db.query("SELECT * FROM users WHERE id=%s", user_id)
    r.setex(key, 3600, json.dumps(user))   # TTL 1 hour
    return user

# Write-Through — write to cache AND DB simultaneously
def update_user(user_id, data):
    db.update(user_id, data)
    r.setex(f"user:{user_id}", 3600, json.dumps(data))

λ Lambda — Serverless Compute

Cold Starts, Concurrency, Destinations
# Cold start optimization
# 1. Use ARM64 (Graviton2) — faster init, cheaper
# 2. Minimize package size — trim dependencies
# 3. Python/Node cold starts faster than Java
# 4. Provisioned Concurrency — pre-warm N instances
# 5. Keep SDK clients OUTSIDE the handler (reused on warm invokes)

import json, boto3

# Initialize outside handler — reused on warm invocations
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('AppTable')

def lambda_handler(event, context):
    try:
        body = json.loads(event.get('body', '{}'))
        result = table.get_item(Key={'PK': f"USER#{body['userId']}"})
        return {
            'statusCode': 200,
            'headers': {'Content-Type': 'application/json'},
            'body': json.dumps(result.get('Item', {}))
        }
    except Exception as e:
        return {'statusCode': 500, 'body': json.dumps({'error': str(e)})}

# Concurrency limits
# Account default: 1000 concurrent executions per region
# Reserved concurrency: guarantee for critical functions
# Provisioned concurrency: always-warm (zero cold start)

# Async invocation destinations
aws lambda put-function-event-invoke-config \
  --function-name MyFunction \
  --destination-config '{
    "OnSuccess": {"Destination": "arn:aws:sqs:::success-queue"},
    "OnFailure": {"Destination": "arn:aws:sqs:::dlq"}
  }'

KMS Envelope Encryption

# CMK (Customer Master Key) → encrypts Data Key → Data Key encrypts data
# CMK never leaves KMS. Only encrypted data key stored with ciphertext.

import boto3
kms = boto3.client('kms')

# Generate data key
response = kms.generate_data_key(
    KeyId='arn:aws:kms:us-east-1:ACCOUNT:key/KEY-ID',
    KeySpec='AES_256'
)
plaintext_key = response['Plaintext']       # use for encryption, then discard
encrypted_key = response['CiphertextBlob']  # store with encrypted data

# Decrypt data key later
response = kms.decrypt(CiphertextBlob=encrypted_key)
plaintext_key = response['Plaintext']

🎯 AWS Interview Questions

Top 10 AWS Interview Q&A

Q1: S3 returns 403. What do you check?

1) Bucket policy — any explicit Deny? 2) IAM policy — does principal have s3:GetObject? 3) Block Public Access settings. 4) S3 Object ACLs. 5) KMS key policy — if encrypted, does IAM have kms:Decrypt? 6) Cross-account — resource-based policy allows? 7) VPC endpoint policy (if using endpoint).

Q2: How does Auto Scaling decide when to scale?

ASG uses scaling policies: Target Tracking (maintain metric at target — CPU=70%), Step Scaling (step adjustments on alarm threshold breaches), Scheduled (pre-warm at 9am), Predictive (ML-based, scales before load arrives). Cooldown periods prevent thrashing.

Q3: Difference between NLB and ALB?

FeatureALB (Layer 7)NLB (Layer 4)
LayerHTTP/HTTPSTCP/UDP/TLS
RoutingPath, host, headers, query stringIP + port only
Static IPNo (use Global Accelerator)Yes (Elastic IP per AZ)
Use forWeb apps, microservicesGaming, IoT, low-latency financial

Q4: How to design for 99.99% availability?

Multi-AZ (3 AZs) + Multi-region (Route53 failover with health checks). Stateless compute (session in Redis/DynamoDB). DB: RDS Multi-AZ or Aurora Global. CDN for static. Circuit breakers for cascades. Chaos testing to verify recovery. RTO/RPO targets define architecture choices.

Q5: Explain DynamoDB hot partition problem

Hot partition occurs when too many reads/writes go to a single partition (e.g., using timestamp or status as PK — low cardinality). Fix: use high-cardinality keys (UUID, userId). For write-heavy: add random suffix to PK (write sharding), combine in GSI for reads.

Q6: How does CloudFront cache invalidation work?

Files cached until TTL expires OR explicit invalidation. Specify path patterns (/images/*). First 1000 paths/month free, then $0.005/path. Best practice: use versioned filenames (app.v2.js) to avoid invalidations — deploy new version, update HTML reference.

Q7: What is the purpose of a NAT Gateway?

Allows instances in private subnets to initiate outbound internet connections (e.g., for OS updates, API calls) without allowing inbound connections from the internet. NAT GW is in a public subnet; private subnet route table sends 0.0.0.0/0 to NAT GW. Managed, scales automatically, $0.045/hour + data processing.

# Kinesis producer — put a single record
import boto3, json, time

kinesis = boto3.client('kinesis', region_name='us-east-1')

record = {
    'user_id': 'u-1234',
    'event': 'purchase',
    'amount': 59.99,
    'timestamp': int(time.time())
}

kinesis.put_record(
    StreamName='user-events',
    Data=json.dumps(record),
    PartitionKey=record['user_id']   # routes to consistent shard
)
# Kinesis consumer — read from a shard
shard_id = 'shardId-000000000000'
resp = kinesis.get_shard_iterator(
    StreamName='user-events',
    ShardId=shard_id,
    ShardIteratorType='TRIM_HORIZON'   # read from oldest record
)
iterator = resp['ShardIterator']

while True:
    result = kinesis.get_records(ShardIterator=iterator, Limit=100)
    for rec in result['Records']:
        data = json.loads(rec['Data'])
        print(data)
    iterator = result['NextShardIterator']
    time.sleep(1)   # poll interval

Key concept: Each shard = 1 MB/s write, 2 MB/s read, 1,000 records/s. Scale by adding shards (split) or reducing (merge).

AWS Glue — Serverless ETL

# Glue PySpark job example
import sys
from awsglue.transforms import *
from awsglue.utils import getResolvedOptions
from pyspark.context import SparkContext
from awsglue.context import GlueContext
from awsglue.job import Job

args = getResolvedOptions(sys.argv, ['JOB_NAME', 'source_bucket', 'target_bucket'])
sc = SparkContext()
glueContext = GlueContext(sc)
spark = glueContext.spark_session
job = Job(glueContext)
job.init(args['JOB_NAME'], args)

# Read from S3 (Glue DynamicFrame)
raw_data = glueContext.create_dynamic_frame.from_options(
    connection_type="s3",
    connection_options={"paths": [f"s3://{args['source_bucket']}/raw/"]},
    format="json"
)

# Apply transformations
from awsglue.transforms import ApplyMapping, DropNullFields
mapped = ApplyMapping.apply(
    frame=raw_data,
    mappings=[
        ("user_id",    "string", "user_id",             "string"),
        ("event_time", "string", "event_timestamp",     "timestamp"),
        ("amount",     "double", "transaction_amount",  "decimal"),
    ]
)
clean = DropNullFields.apply(frame=mapped)

# Write to Parquet in S3 (partitioned)
glueContext.write_dynamic_frame.from_options(
    frame=clean,
    connection_type="s3",
    connection_options={
        "path": f"s3://{args['target_bucket']}/curated/",
        "partitionKeys": ["year", "month", "day"]
    },
    format="parquet"
)
job.commit()

Amazon Athena — Serverless SQL on S3

-- Create external table over S3 Parquet data
CREATE EXTERNAL TABLE transactions (
    user_id     STRING,
    event_time  TIMESTAMP,
    amount      DECIMAL(10,2),
    category    STRING
)
PARTITIONED BY (year INT, month INT, day INT)
STORED AS PARQUET
LOCATION 's3://my-data-lake/curated/transactions/'
TBLPROPERTIES ('parquet.compress'='SNAPPY');

-- Load partitions (or use MSCK REPAIR TABLE)
ALTER TABLE transactions ADD PARTITION (year=2024, month=1, day=15)
LOCATION 's3://my-data-lake/curated/transactions/year=2024/month=1/day=15/';

-- Query with partition pruning (efficient — scans only relevant partitions)
SELECT category, SUM(amount) AS total, COUNT(*) AS tx_count
FROM transactions
WHERE year = 2024 AND month = 1
GROUP BY category
ORDER BY total DESC;

-- Cost optimization: always filter on partition keys, use columnar format
-- $5 per TB scanned — partitioning can reduce cost 99%

MLOps with SageMaker Pipelines

from sagemaker.workflow.pipeline import Pipeline
from sagemaker.workflow.steps import ProcessingStep, TrainingStep, RegisterModel
from sagemaker.workflow.conditions import ConditionGreaterThanOrEqualTo
from sagemaker.workflow.condition_step import ConditionStep

# Define pipeline steps
step_process = ProcessingStep(name="Preprocessing", ...)
step_train   = TrainingStep(name="Training", depends_on=[step_process], ...)

# Conditional registration — only register if accuracy >= 0.85
cond_acc = ConditionGreaterThanOrEqualTo(
    left=JsonGet(step_name=step_eval.name, property_file=eval_report, json_path="metrics.accuracy.value"),
    right=0.85
)
step_cond = ConditionStep(
    name="CheckAccuracy",
    conditions=[cond_acc],
    if_steps=[step_register],
    else_steps=[step_fail]
)

pipeline = Pipeline(
    name="FraudDetectionPipeline",
    steps=[step_process, step_train, step_eval, step_cond],
)
pipeline.upsert(role_arn=role)
pipeline.start()
