IAM — Identity & Access

Global Service
Core Concepts
UsersPeople or services with credentials
GroupsCollection of users, assign policies
RolesTemporary permissions, assume by services
PoliciesJSON rules — Allow / Deny on actions
Key Rules
Least Privilege: Always give minimum required permissions only.
Root account: Never use daily. Enable MFA immediately. Create admin IAM user instead.
Access Keys: For CLI/SDK. Never share. Rotate regularly.
Policy Evaluation
Explicit DenyALWAYS wins over Allow
Explicit AllowGrants access if no deny
Implicit DenyDefault — nothing allowed

Operational Excellence
Run & monitor systems. Automate changes. Respond to events. IaC, CI/CD, small changes.
Security
Protect data, systems, assets. IAM least privilege. Encrypt data. Enable traceability.
Reliability

Performance Efficiency
Use resources efficiently. Right instance types. Serverless. Global distribution via CDN.

Minimize environmental impact. Use managed services, efficient hardware, reduce waste.

Common Exam Traps

Multi-AZ ≠ performance boost. Multi-AZ is standby ONLY — no reads served from standby (except Aurora). Use Read Replicas for performance.
S3 is NOT a file system. It's object storage. Cannot mount like EFS. Cannot use for OS boot disk.
Instance Store data LOST on stop/terminate. Never store permanent data on Instance Store.
SQS Standard = at least once. Your app must handle duplicate messages. Use FIFO if exactly-once needed.
NACLs are STATELESS. Must add both inbound AND outbound rules for return traffic.
Security Groups are STATEFUL. Return traffic auto-allowed. Only need inbound rule.
Direct Connect ≠ instant. Takes weeks to provision. Not a DR quick solution.
CloudFront caches at Edge. Invalidation = $. To update content, use versioned file names instead.
Reserved Instances apply to any instance matching the type/region — not locked to 1 specific instance.
Lambda max 15 min. For longer jobs use ECS/Fargate, Step Functions, or Batch.
