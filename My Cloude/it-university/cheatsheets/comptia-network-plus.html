IPv6 Addressing

128-bit · 8 groups of 4 hex
Address Types
UnicastSingle interface — one-to-one delivery
MulticastFF00::/8 — one-to-many (group)
AnycastNearest interface in group
Address Ranges
::1Loopback (= 127.0.0.1 in IPv4)
fe80::/10Link-local — auto-configured, non-routable
2000::/3Global unicast — public internet
fc00::/7Unique local (ULA) — private, FD00::/8 typical
FF00::/8Multicast — all routers FF02::2
FF02::1All nodes multicast
FF02::2All routers multicast
EUI-64 Interface ID
Source48-bit MAC address
Step 1Split MAC in half, insert FF:FE in middle
Step 2Flip bit 7 (U/L bit) of first octet
Result64-bit interface identifier appended to /64 prefix
Notation Rules
Leading zerosCan be omitted per group (0DB8 → DB8)
::Replaces one consecutive run of zero groups (once only)
Transition Technologies
Dual StackIPv4 + IPv6 simultaneously on device

TeredoIPv6 over IPv4 UDP · prefix 2001::/32

Switching & VLANs

Layer 2 Technologies
VLANs (802.1Q)
RangeVLAN 1–4094 · VLAN 1 = default native VLAN
PurposeLogical segmentation · reduces broadcast domain
802.1Q Tag4-byte tag inserted in Ethernet frame (TPID + TCI)
Trunk PortCarries multiple VLANs · connects switches/routers
Access PortSingle VLAN · connects end devices
Voice VLANSeparate VLAN for VoIP traffic (QoS priority)
STP — Spanning Tree Protocol (802.1D)
PurposePrevents Layer 2 loops in redundant topology
Root BridgeLowest BID = priority (default 32768) + MAC address
Port StatesBlocking → Listening → Learning → Forwarding → Disabled
Convergence~50 seconds (802.1D) — slow
RSTP (802.1w) — Rapid STP
Convergence~1–2 seconds · backward compatible with STP
Port RolesRoot · Designated · Alternate · Backup
Port StatesDiscarding · Learning · Forwarding (3 states only)
Port Security
Max MACsLimit number of MAC addresses per port
ProtectDrops traffic from unknown MACs, no notification
RestrictDrops + logs + increments violation counter
ShutdownPuts port in err-disabled state (default mode)
EtherChannel / LACP
LACP802.3ad · open standard · active/passive modes
PAgPCisco proprietary · desirable/auto modes
BenefitLink aggregation — bandwidth + redundancy
BPDU Guard: disables port if STP BPDU received on portfast port

Troubleshooting Methodology

CompTIA 7-Step Process
CompTIA Official Steps
Step 1Identify the problem — gather info, symptoms, duplicate
Step 2Establish a theory of probable cause
Step 3Test the theory to determine cause
Step 4Establish a plan of action & identify potential effects
Step 5Implement the solution or escalate
Step 6Verify full system functionality + preventive measures
Step 7Document findings, actions, and outcomes
OSI Troubleshooting Approaches
Bottom-UpLayer 1 first → Physical → check cables, LEDs, speeds
Top-DownLayer 7 first → Application → check configs, connectivity
Divide & ConquerStart at middle layer → ping test (Layer 3)
Common Issues by Layer
L1 PhysicalDamaged cable · wrong speed/duplex · bad NIC · no link light
L2 Data LinkDuplex mismatch · wrong VLAN · STP loop · MAC table full
L3 NetworkWrong IP/mask/GW · routing loop · missing route
L4 TransportFirewall blocking port · wrong port config · TCP reset
L7 ApplicationDNS failure · wrong URL · auth error · service down
Duplex & Speed Issues
Half duplexOne direction at a time · CSMA/CD · hubs
Full duplexSimultaneous bidirectional · switches · no collisions
Duplex mismatchHigh collisions · late collisions · poor performance
CRC errors = physical layer issue · late collisions = duplex mismatch · runts = undersized frames
