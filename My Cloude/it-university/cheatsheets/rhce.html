Playbook Structure

Annotated Example
Full Annotated Playbook
--- - name: Configure web servers   hosts: webservers   become: true   vars:     http_port: 80     doc_root: /var/www/html   pre_tasks:     - name: Ensure up to date       yum:         name: "*"         state: latest   roles:     - common     - { role: apache, when: "ansible_os_family == 'RedHat'" }   tasks:     - name: Install httpd       yum:         name: httpd         state: present       tags: packages     - name: Start httpd       service:         name: httpd         state: started         enabled: true       notify: Restart firewall   handlers:     - name: Restart firewall       service:         name: firewalld         state: restarted   post_tasks:     - name: Verify port open       wait_for:         port: "{{ http_port }}"         timeout: 30
Order: pre_tasks → roles → tasks → post_tasks. Handlers run after each section.

Conditionals & Loops

when: + loop:
Conditionals (when:)
when: ansible_os_family == "RedHat" when: ansible_distribution_version is version('8', '>=') when: result.rc != 0 when: "'httpd' in ansible_facts.packages" when: myvar is defined when: myvar is not defined when: inventory_hostname in groups['webservers'] when: # AND logic (all must be true)   - ansible_os_family == "RedHat"   - ansible_distribution_version is version('8', '>=')
Loops
loop: # simple list loop   - httpd   - firewalld   - python3 # access with: {{ item }}   loop: "{{ packages_list }}" # from variable   loop: "{{ users | dict2items }}" # Dict loop # access: {{ item.key }}, {{ item.value }}   with_items: # older style, still works   - httpd   - nginx
Loop Control
loop_control:   label: "{{ item.name }}" # cleaner output   index_var: idx # loop index (0-based)   loop_var: pkg # rename 'item'   pause: 2 # seconds between iterations
Use loop: (not with_items) for new playbooks. loop_control.label prevents huge output when looping over dicts.

Critical Modules

File, Package, User, System, Storage
File Management
copy: src=files/httpd.conf dest=/etc/httpd/conf/httpd.conf owner=root mode='0644' backup=true template: src=templates/vhost.j2 dest=/etc/httpd/conf.d/vhost.conf mode='0644' file: path=/var/www/html state=directory owner=apache mode='0755' recurse=true lineinfile: path=/etc/ssh/sshd_config regexp='^PermitRootLogin' line='PermitRootLogin no' backup=true blockinfile: path=/etc/hosts block="192.168.1.10 web1\n192.168.1.20 web2" marker="# {mark} ANSIBLE MANAGED" replace: path=/etc/httpd/conf/httpd.conf regexp='Listen 80' replace='Listen {{ http_port }}'

User Management

Roles

Structure + Galaxy + Collections
Create Role Structure
ansible-galaxy init myrole # Creates: # myrole/ # tasks/main.yml ← task list # handlers/main.yml ← handlers # vars/main.yml ← high-priority vars # defaults/main.yml ← low-priority defaults # files/ ← static files (copy module) # templates/ ← Jinja2 templates # meta/main.yml ← dependencies, metadata # README.md
Using Roles in Playbooks
roles: # in roles: section   - common   - role: apache     vars:       http_port: 443     when: ansible_os_family == "RedHat"   - include_role: # dynamic (loop/when OK)     name: apache   - import_role: # static (parsed at load)     name: common
Role Dependencies (meta/main.yml)
dependencies:   - role: common     vars:       some_var: value   - role: firewall
Galaxy Collections
ansible-galaxy collection install community.general ansible-galaxy collection install ansible.posix ansible-galaxy collection list # Use in playbook: collections:   - community.general   - ansible.posix
import vs include: import_ = static (compile time). include_ = dynamic (runtime). Use import_ unless you need loops/conditionals on the include itself.

Ansible Vault

Encrypt Sensitive Data
Vault Operations
ansible-vault create secrets.yml # create new encrypted file ansible-vault encrypt existing.yml # encrypt existing file ansible-vault decrypt existing.yml # decrypt (careful!) ansible-vault edit secrets.yml # edit in-place ansible-vault view secrets.yml # view without decrypting ansible-vault rekey secrets.yml # change password   # Encrypt a single string value: ansible-vault encrypt_string 'MyP@ssw0rd' --name 'db_password' # → outputs: db_password: !vault | # $ANSIBLE_VAULT;1.1;AES256 # ...
Using Vault in Playbooks
# Option 1: prompt at runtime ansible-playbook site.yml --ask-vault-pass   # Option 2: password file (exam preferred) ansible-playbook site.yml --vault-password-file ~/.vault_pass   # Option 3: multiple vault IDs ansible-playbook site.yml \   --vault-id prod@~/.vault_prod \   --vault-id dev@~/.vault_dev
Store Encrypted Vars in group_vars
# group_vars/all/vault.yml (encrypted) db_password: MySecretPassword api_key: abc123xyz   # group_vars/all/vars.yml (plaintext) db_host: "{{ vault_db_host }}"
Best Practices
Namingvault_prefix: vault_db_pass vs db_pass
.gitignoreAdd vault password file to .gitignore
Rotationansible-vault rekey to rotate passwords
CommitNever commit unencrypted secrets
Exam critical: The vault password file must exist and be readable. Use chmod 600 ~/.vault_pass. Always test with --ask-vault-pass first.

Error Handling & Tags

Block/Rescue/Always + Check Mode
Block / Rescue / Always
tasks:   - block:       - name: Try to install package         yum:           name: mypackage           state: present       - name: Start service         service:           name: myservice           state: started     rescue:       - name: Handle failure         debug:           msg: "Failed: {{ ansible_failed_result }}"     always:       - name: Always clean up         file: path=/tmp/install.lock state=absent
Error Control Keywords
ignore_errors: truecontinue on failure
failed_when: result.rc > 1custom failure condition
changed_when: falsenever report changed
any_errors_fatal: truestop all hosts on any failure
max_fail_percentage: 30fail if >30% of hosts fail
Tags
tasks:   - name: Install packages     yum: name=httpd state=present     tags: [packages, setup]   - name: Configure     template: src=httpd.j2 dest=/etc/httpd.conf     tags: configure   ansible-playbook site.yml --tags packages,configure ansible-playbook site.yml -t packages ansible-playbook site.yml --skip-tags configure
Special Tags
alwaysalways runs (even without --tags)
neveronly runs if explicitly specified
taggedrun all tagged tasks
untaggedrun all untagged tasks
allrun everything (default)
Check & Diff Mode
ansible-playbook site.yml --check # dry run ansible-playbook site.yml --diff # show file diffs ansible-playbook site.yml --check --diff # both together ansible-playbook site.yml --list-tasks # list what would run ansible-playbook site.yml --list-hosts # list targeted hosts ansible-playbook site.yml --list-tags # list all tags

Common Exam Tasks — EX294 Practice Checklist

Exact task types that appear on EX294 — practice until automatic
1. Configure Control Node
dnf install ansible-core # Set up ansible.cfg + inventory # Test: ansible all -m ping
2. Deploy Packages Across Host Groups
- yum: name={{ item }} state=present   loop: "{{ required_packages }}"
3. Create Users with Hashed Passwords
vars:   users:     - name: alice       password: SecretPass1 tasks:   - user:       name: "{{ item.name }}"       password: "{{ item.password | password_hash('sha512') }}"       update_password: on_create     loop: "{{ users }}"
4. Configure Service + Firewall + SELinux
- service: name=httpd state=started enabled=true - firewalld: service=http permanent=true immediate=true state=enabled - seboolean: name=httpd_can_network_connect state=true persistent=true
5. Create Role and Use It
ansible-galaxy init roles/myrole # write tasks/main.yml, templates/, defaults/main.yml # reference in playbook: roles: [myrole]
6. Encrypt with Vault and Use in Play
ansible-vault create group_vars/all/vault.yml # add: vault_db_pass: secret # reference in play: "{{ vault_db_pass }}" ansible-playbook site.yml --vault-password-file ~/.vault_pass
7. NFS Mount
mount:   path: /mnt/nfs   src: nfs-server:/exports/data   fstype: nfs   opts: "ro,vers=4"   state: mounted
8. LVM + Filesystem + Mount
- lvol: vg=vg0 lv=data size=2g - filesystem: fstype=ext4 dev=/dev/vg0/data - mount: path=/data src=/dev/vg0/data fstype=ext4 state=mounted
9. Cron Job
cron:   name: "System backup"   minute: "30"   hour: "23"   job: "/usr/local/bin/backup.sh > /var/log/backup.log 2>&1"
10. Use Facts in Template (MOTD)
# In template (motd.j2): Welcome to {{ ansible_hostname }} OS: {{ ansible_distribution }} {{ ansible_distribution_version }} IP: {{ ansible_default_ipv4.address }}   # In playbook: - template: src=motd.j2 dest=/etc/motd
Final exam tips: Always verify your playbook runs without errors on all hosts. Use --check --diff to preview changes. Check idempotency — running twice should show 0 changes. Score is binary per task: it either works or it doesn't.
