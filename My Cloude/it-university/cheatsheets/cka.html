NetworkPolicy

Services & Networking 20%
Default: all traffic allowed. NetworkPolicy requires a CNI that supports it — Calico, Cilium, or Weave (not Flannel alone).
NetworkPolicy YAML structure
apiVersion: networking.k8s.io/v1 kind: NetworkPolicy metadata:   name: allow-frontend   namespace: default spec:   podSelector:     matchLabels: {app: backend}   policyTypes: [Ingress, Egress]   ingress:   - from:     - podSelector:         matchLabels: {app: frontend}     - namespaceSelector:         matchLabels: {name: prod}     ports:     - protocol: TCP       port: 8080   egress:   - to:     - ipBlock:         cidr: 10.0.0.0/8     ports:     - port: 5432
Common patterns
Deny all ingresspodSelector:{} ingress:[]
Deny all egresspodSelector:{} egress:[]
Allow all ingressingress: [{}]
Allow all egressegress: [{}]
Selector types in from/to
podSelectorMatch pods by label (same ns)
namespaceSelectorMatch entire namespaces
ipBlockCIDR ranges, optional except
Multiple from items in the same rule are OR logic. Separate - list items under from are AND when on same item.

Cluster Upgrade (kubeadm)

Cluster Architecture 25%
Upgrade one minor version at a time. Always upgrade control plane first, then workers.
1. Control plane — check & upgrade kubeadm
kubeadm upgrade plan apt-get update apt-get install -y kubeadm=1.28.0-00 kubeadm upgrade apply v1.28.0 kubeadm upgrade apply v1.28.0 --dry-run
2. Drain control plane node
k drain controlplane --ignore-daemonsets
3. Upgrade kubelet + kubectl on control plane
apt-get install -y kubelet=1.28.0-00 kubectl=1.28.0-00 systemctl daemon-reload systemctl restart kubelet
4. Uncordon control plane
k uncordon controlplane k get nodes
5. Worker node upgrade (per worker)
# From control plane: k drain worker1 --ignore-daemonsets --delete-emptydir-data # SSH into worker: ssh worker1 apt-get update apt-get install -y kubeadm=1.28.0-00 kubeadm upgrade node apt-get install -y kubelet=1.28.0-00 kubectl=1.28.0-00 systemctl daemon-reload systemctl restart kubelet exit # Back on control plane: k uncordon worker1
Verify upgrade
k get nodes kubelet --version kubeadm version

Time Management
17 tasks in 2 hours = ~7 min each. Skip and flag tasks that are taking too long — return at end. etcd backup and cluster upgrade tasks are worth most points.
Verify Your Work
After every task run k get <resource> or k describe to confirm the resource is correct. A pod in Pending/Error state is not a passing answer.
Use kubectl explain
k explain <resource>.spec.field --recursive — faster than searching docs for field names and nested structure.
Namespace Matters
Every task specifies a namespace. Always add -n <ns> or set the current namespace. Resources in wrong namespace = wrong answer.
