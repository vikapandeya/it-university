STP / RSTP

802.1D / 802.1w
STP 802.1D — Root Bridge Election
Bridge IDPriority (0–65535) + MAC address
Default Priority32768 (+ VLAN ID in PVST+)
Root BridgeLowest Bridge ID wins
STP Port States
Blocking20s — receives BPDUs only
Listening15s — forward delay, no learning
Learning15s — builds MAC table
ForwardingActive — passes traffic
DisabledAdmin shut or STP disabled
STP Timers
Hello2 seconds
Max Age20 seconds
Forward Delay15 seconds (×2 = 30s total)
STP Port Roles
Root Port (RP)Best path toward root bridge
Designated Port (DP)Best port on each segment
Non-DesignatedBlocking — prevents loops
RSTP 802.1w
Port StatesDiscarding → Learning → Forwarding
Extra RolesAlternate (backup RP), Backup (backup DP)
Convergence<1 second (proposal/agreement)
STP Protection Features
PortFastSkips L/L states — access ports only
BPDUGuardErr-disables port if BPDU received
BPDUFilterStops sending/receiving BPDUs
Root GuardPrevents superior BPDU takeover

EIGRP

Enhanced Interior Gateway Routing
Key Characteristics
TypeCisco hybrid (advanced distance vector)
AD Internal90
AD External170
AD Summary5
ProtocolIP protocol 88
Multicast224.0.0.10
Metric Components (K-values)
Default FormulaK1×BW + K3×Delay
K1, K3= 1 (Bandwidth + Delay used)
K2, K4, K5= 0 (Load, Reliability unused)
BW component10^7 / min-BW-in-Kbps × 256
DUAL Algorithm
SuccessorBest path — installed in routing table
Feasible SuccessorBackup path meeting FC
FDFeasible Distance — metric from local to dest
RD / ADReported Distance — neighbor's metric to dest
Feasibility ConditionRD < FD of current successor
PurposeGuarantees loop-free alternate path
EIGRP Tables
Neighbor TableAdjacent EIGRP routers
Topology TableAll routes received from neighbors
Routing TableBest routes (successors)
Neighbor Requirements
ASNSame Autonomous System Number
K-valuesMust match
SubnetSame subnet on connecting interface
Named EIGRP mode: router eigrp [name] — supports address-family ipv4/ipv6.

FHRP — First Hop Redundancy

HSRP / VRRP / GLBP
HSRP — Hot Standby Router Protocol (Cisco)
RolesActive + Standby + Listen
VIPVirtual IP — default gateway for hosts
TimersHello 3s / Hold 10s
Groups0–255 (HSRPv1), 0–4095 (HSRPv2)
PreemptDisabled by default — must configure
Virtual MAC0000.0c07.acXX (XX = group hex)
ElectionHighest priority (default 100), then highest IP
VRRP — Virtual Router Redundancy (Open)
RolesMaster + Backup(s)
Groups1–255
TimersHello 1s / Master down ~3s
PreemptEnabled by default
Virtual MAC0000.5e00.01XX (XX = group hex)
GLBP — Gateway Load Balancing (Cisco)
RolesAVG (Active Virtual Gateway) + AVFs
AdvantageLoad balancing — multiple gateways active
AVGAssigns virtual MACs to AVFs
LB MethodsRound-robin (default), weighted, host-dependent
HSRP = Cisco only. VRRP = open standard. GLBP = Cisco only with true load balancing.

QoS

Quality of Service
QoS Steps
1. ClassificationIdentify traffic (ACL, NBAR, DSCP)
2. MarkingTag packets (DSCP, CoS, IP Precedence)
3. QueuingBuffer and schedule packets
4. Congestion AvoidanceWRED — proactive drops
5. Policing/ShapingEnforce rate limits
DSCP Values (6-bit field)
EF (46)Expedited Forwarding — VoIP bearer
AF41/42/43Video conferencing
AF31/32/33Call signaling
CS3Signaling
BE / CS0 (0)Best Effort — default
CoS (Class of Service)
Location3-bit PCP field in 802.1Q tag
Range0–7 (7 = highest priority)
CoS 5VoIP
CoS 4Video
Queuing Mechanisms
FIFOFirst In First Out — no QoS
WFQWeighted Fair Queuing — automatic
CBWFQClass-Based WFQ — user-defined classes
LLQLow Latency Queue — strict priority for voice
Traffic Management
PolicingDrops or re-marks excess traffic (hard limit)
ShapingBuffers/delays excess traffic (smooth)
Voice needs LLQ: jitter <30ms, delay <150ms, loss <1% for G.711/G.729.
