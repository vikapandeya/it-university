Pod & Workload Objects

core specs
Pod Spec Essentials
apiVersion: v1 kind: Pod metadata:   name: myapp   labels:     app: myapp     tier: frontend spec:   containers:   - name: main     image: nginx:1.25     ports:     - containerPort: 80     resources:       requests: {cpu: 100m, memory: 128Mi}       limits: {cpu: 500m, memory: 512Mi}     env:     - name: ENV_VAR       value: "production"   restartPolicy: Always # Always/OnFailure/Never
Deployment (Rolling Strategy)
apiVersion: apps/v1 kind: Deployment spec:   replicas: 3   selector: {matchLabels: {app: myapp}}   strategy:     type: RollingUpdate     rollingUpdate: {maxSurge: 1, maxUnavailable: 0}   template:     metadata: {labels: {app: myapp}}     spec:       containers:       - name: myapp         image: nginx:1.25
StatefulSet / DaemonSet / Job / CronJob
StatefulSetordered pod-0,pod-1; stable network IDs; PV per pod; databases
DaemonSetone pod per node; no replicas field; log collectors/node agents
Jobrun-to-completion; completions/parallelism/backoffLimit/activeDeadlineSeconds
CronJobschedule (cron syntax); successfulJobsHistoryLimit; concurrencyPolicy: Forbid/Allow/Replace

Multi-Container Patterns

sidecar / adapter / ambassador / init
Sidecar Pattern
spec:   containers:   - name: main-app     image: myapp:latest     volumeMounts:     - name: logs       mountPath: /var/log/app   - name: log-shipper # sidecar     image: fluentd:latest     volumeMounts:     - name: logs       mountPath: /var/log/app       readOnly: true   volumes:   - name: logs     emptyDir: {}
Init Containers
initContainers: - name: wait-for-db   image: busybox   command: ['sh','-c','until nc -z db-svc 5432; do sleep 2; done'] - name: db-migrate   image: myapp:latest   command: ['python', 'manage.py', 'migrate'] containers: - name: myapp   image: myapp:latest
Pattern Summary
Sidecarauxiliary container sharing pod lifecycle (log shippers, metrics exporters)
Adaptertransforms/normalizes output from main container before sending to external system
Ambassadorproxy for outbound requests, handles service discovery/routing (Envoy sidecar)
Initsequential startup tasks — must complete before app starts, each runs to completion
Init containers run sequentially and must succeed before any app container starts. Use them for DB migrations, config fetching, or dependency checks.

Probes & Lifecycle

liveness / readiness / startup / hooks
Probe Types
livenessProbe: # restart container if fails   httpGet: {path: /healthz, port: 8080}   initialDelaySeconds: 15 # wait before first check   periodSeconds: 10 # check interval   failureThreshold: 3 # fails before restart   successThreshold: 1   timeoutSeconds: 5   readinessProbe: # remove from endpoints if fails   exec:     command: [cat, /tmp/healthy]   initialDelaySeconds: 5   periodSeconds: 5   startupProbe: # protect slow-starting containers   tcpSocket: {port: 8080}   failureThreshold: 30 # 30 * 10s = 5 min max startup   periodSeconds: 10
Probe Methods
httpGetHTTP GET to path/port — 200-399 = success
execCommand exits 0 = success
tcpSocketTCP connection established = success
grpcgRPC health check (K8s 1.24+)
Lifecycle Hooks
lifecycle:   postStart: # runs immediately after container starts     exec:       command: ["/bin/sh","-c","echo started > /tmp/started"]   preStop: # runs before SIGTERM is sent     exec:       command: ["/bin/sh","-c","/usr/sbin/nginx -s quit"] terminationGracePeriodSeconds: 30 # default 30s
startupProbe disables liveness/readiness until it succeeds. Use when app takes >initialDelaySeconds to start. Prevents premature restarts.

Use in Pod — Env Var

Use in Pod — Volume Mount
volumes: - name: config-vol   configMap:     name: myconfig     items:     - key: nginx.conf       path: nginx.conf volumeMounts: - name: config-vol   mountPath: /etc/nginx

Secrets are base64 encoded, NOT encrypted by default. Enable EncryptionConfiguration for etcd encryption at rest.

Resource Management

requests / limits / QoS / quota
Requests & Limits
resources:   requests:     cpu: "250m" # 0.25 core — scheduled on node with this available     memory: "256Mi" # scheduled on node with this available   limits:     cpu: "1000m" # throttled (not killed) at this     memory: "512Mi" # killed (OOMKilled) if exceeded
QoS Classes
Guaranteedrequests == limits for ALL containers → highest priority, last evicted
Burstableat least one request set, but req != limit → middle priority
BestEffortno requests or limits at all → lowest priority, first evicted
LimitRange (per-namespace defaults)
apiVersion: v1 kind: LimitRange spec:   limits:   - type: Container     default: {memory: 512Mi, cpu: 500m}     defaultRequest: {memory: 256Mi, cpu: 100m}     max: {memory: 2Gi}     min: {memory: 64Mi}
ResourceQuota (namespace-wide)
spec:   hard:     pods: "20"     requests.cpu: "4"     requests.memory: 4Gi     limits.cpu: "8"     limits.memory: 8Gi     count/deployments.apps: "10"
Commands
k top pods -n myns # requires metrics-server k top nodes k describe quota -n myns k describe limitrange -n myns

Deployment Strategies

rolling / recreate / rollback / HPA
Rolling Update (default)
strategy:   type: RollingUpdate   rollingUpdate:     maxSurge: 25% # extra pods allowed above desired     maxUnavailable: 25% # max pods below desired during update
Recreate Strategy
strategy:   type: Recreate # kills ALL old pods, then creates new                          # causes downtime; use for stateful apps
Rollout Commands
k rollout status deploy/myapp # watch rollout progress k rollout history deploy/myapp # show revision history k rollout history deploy/myapp --revision=2 k rollout undo deploy/myapp # roll back to previous k rollout undo deploy/myapp --to-revision=2 k rollout pause deploy/myapp # pause rollout k rollout resume deploy/myapp # resume paused rollout k set image deploy/myapp app=nginx:1.26 # trigger update
Scaling & HPA
k scale deploy myapp --replicas=5 k autoscale deploy myapp --min=2 --max=10 --cpu-percent=70 # Creates HPA — requires metrics-server   apiVersion: autoscaling/v2 kind: HorizontalPodAutoscaler spec:   scaleTargetRef:     apiVersion: apps/v1     kind: Deployment     name: myapp   minReplicas: 2   maxReplicas: 10   metrics:   - type: Resource     resource:       name: cpu       target: {type: Utilization, averageUtilization: 70}
Annotate deployments to track change cause: k annotate deploy myapp kubernetes.io/change-cause="update image to 1.26" — visible in rollout history.

Security Context

pod / container / RBAC / SA
Pod-Level vs Container-Level
spec:   securityContext: # pod-level — all containers     runAsUser: 1000     runAsGroup: 3000     fsGroup: 2000 # volume ownership     runAsNonRoot: true     sysctls:     - name: net.core.somaxconn       value: "1024"   containers:   - name: app     securityContext: # container-level (overrides pod)       runAsUser: 2000       allowPrivilegeEscalation: false       readOnlyRootFilesystem: true       capabilities:         drop: [ALL]         add: [NET_BIND_SERVICE] # port < 1024       seccompProfile: {type: RuntimeDefault}
ServiceAccount
k create serviceaccount myapp-sa   spec:   serviceAccountName: myapp-sa   automountServiceAccountToken: false # disable if not needed   volumes: # projected token with expiry - name: token   projected:     sources:     - serviceAccountToken:         path: token         expirationSeconds: 3600         audience: myapp
RBAC for ServiceAccount
k create role pod-reader --verb=get,list --resource=pods k create rolebinding myapp-pod-reader \   --role=pod-reader \   --serviceaccount=default:myapp-sa
Role = namespace-scoped. ClusterRole = cluster-wide. Use RoleBinding to bind ClusterRole in a namespace (limits scope).

Services & Networking

service types / ingress / networkpolicy / DNS
Service Types
# ClusterIP (default) — internal only k expose deploy myapp --port=80 --target-port=8080   # NodePort — external via node IP:port (30000-32767) k expose deploy myapp --type=NodePort --port=80   # LoadBalancer — cloud provider LB k expose deploy myapp --type=LoadBalancer --port=80   # ExternalName — CNAME alias kind: Service spec:   type: ExternalName   externalName: my-database.corp.example.com
Ingress
apiVersion: networking.k8s.io/v1 kind: Ingress metadata:   annotations:     nginx.ingress.kubernetes.io/rewrite-target: / spec:   tls:   - hosts: [myapp.example.com]     secretName: myapp-tls   rules:   - host: myapp.example.com     http:       paths:       - path: /api         pathType: Prefix         backend: {service: {name: api-svc, port: {number: 80}}}       - path: /         pathType: Prefix         backend: {service: {name: frontend-svc, port: {number: 80}}}
NetworkPolicy
kind: NetworkPolicy spec:   podSelector: {matchLabels: {app: backend}}   policyTypes: [Ingress, Egress]   ingress:   - from:     - podSelector: {matchLabels: {app: frontend}}     - namespaceSelector: {matchLabels: {env: production}}     ports: [{protocol: TCP, port: 8080}]   egress:   - to:     - ipBlock: {cidr: 10.0.0.0/8}     ports: [{port: 5432}] # postgres
DNS Patterns
Service DNSsvc-name.namespace.svc.cluster.local
Pod DNSpod-ip-dashes.namespace.pod.cluster.local
NetworkPolicy is additive — once any policy selects a pod, all non-selected traffic is denied. Empty ingress/egress = deny all for that type.

Observability & Debugging

logs / debug / pod states / conditions
Logs
k logs mypod # current logs k logs mypod -c mycontainer # specific container k logs mypod --previous # previous (after crash) k logs mypod -f # follow/stream k logs mypod --since=1h # last hour k logs mypod --since-time=2024-01-01T00:00:00Z k logs mypod --tail=100 # last 100 lines k logs deploy/myapp # any pod in deploy k logs -l app=myapp --all-containers # all matching pods
Debug Commands
k describe pod mypod # Events section most useful k describe node mynode k get events -n myns --sort-by=.lastTimestamp k exec -it mypod -- /bin/sh k exec -it mypod -c sidecar -- bash k debug mypod -it --image=busybox # ephemeral debug k debug node/mynode -it --image=ubuntu # node debug k port-forward pod/mypod 8080:80 k port-forward svc/mysvc 8080:80
Pod States
Pendingnot yet scheduled or pulling image
Runningat least one container running
Succeededall containers exited 0
Failedcontainers exited non-zero
Unknowncannot reach kubelet
CrashLoopBackOffcontainer keeps crashing — check logs --previous
ImagePullBackOffcannot pull image — check image name/credentials
Pod Conditions (k describe)
PodScheduledassigned to node
Initializedall init containers done
ContainersReadyall containers ready
Readypod ready to serve traffic
CrashLoopBackOff — always run k logs mypod --previous to see logs from the crashed container instance.
Pending forever? Check k describe pod Events — common causes: insufficient CPU/memory, no matching node selector, PVC not bound.
ImagePullBackOff — verify image name & tag. For private registry, check imagePullSecrets on pod spec or default service account.
Ephemeral containers via k debug attach a debug container to a running pod without restarting it. Useful when pod has no shell.
