RHCSA EX200 Cheat Sheet

Red Hat Certified System Administrator — RHEL 9 | Performance-Based Lab Exam | All 16 Task Domains

RHEL 9 2.5 Hours Pass: 210/300
Exam Format
Performance-based lab — no multiple choice. You must DO the tasks on a live RHEL 9 system.
Key Habit
systemctl enable --now starts AND persists a service. Don't forget --now or services won't be running at reboot check.
SELinux
Never disable SELinux. Use restorecon or semanage fcontext + restorecon for persistent context fixes.
Firewall
Always use --permanent flag then --reload or your firewall rules vanish on reboot.

Permissions & ACLs

chmod / chown / umask / setfacl
Basic Permissions
chmod 755 file chmod 644 file chmod 600 file chmod 777 file chmod u+x file chmod g-w file chmod o=r file chmod -R 755 /dir
Ownership
chown user:group file chown -R user:group /dir chgrp groupname file
Umask
umask 022files:644 dirs:755 (default)
umask 027files:640 dirs:750
umask 077files:600 dirs:700
umask subtracts from 666 (files) / 777 (dirs). Set in ~/.bashrc or /etc/profile.
Special Bits
chmod u+s fileSUID (4xxx) — run as owner
chmod g+s dirSGID (2xxx) — inherit group
chmod +t /dirSticky (1xxx) — own files only
ls -la /dir
SUID: -rws------ SGID: drwxr-sr-x Sticky: drwxrwxrwt
ACLs
setfacl -m u:user:rwx /path setfacl -m g:group:r-- /path setfacl -R -m u:user:rwx /dir getfacl /path setfacl -b /path setfacl -x u:user /path setfacl -m d:u:user:rwx /dir
Default ACLs (d: prefix) apply to new files created inside the directory. Requires getfacl to verify.

Storage & Partitioning

fdisk / gdisk / mkfs / fstab
Partition Tools
fdisk /dev/sdb gdisk /dev/sdb
n — new partition p — print table d — delete w — write & exit q — quit no save t — change type
partprobe /dev/sdb
Format Filesystems
mkfs.xfs /dev/sdb1 mkfs.ext4 /dev/sdb1 mkfs.xfs -L "datalabel" /dev/sdb1
Inspect Devices
lsblk lsblk -f blkid blkid /dev/sdb1 df -hT findmnt
Mount / Unmount
mount /dev/sdb1 /mnt mount -o ro,noexec /dev/sdb1 /mnt mount -t xfs /dev/sdb1 /mnt umount /mnt umount -l /mnt
/etc/fstab — Persistent Mounts
UUID=xxxx-xxxx /data xfs defaults 0 0 UUID=xxxx-xxxx /data ext4 defaults 0 2 mount -a
Always use UUID (from blkid) in /etc/fstab — device names like /dev/sdb1 can change after reboot. A bad fstab entry can prevent boot.
Swap Partition
mkswap /dev/sdb2 swapon /dev/sdb2 swapoff /dev/sdb2 swapon -s free -h UUID=xxxx swap swap defaults 0 0
Swap File
dd if=/dev/zero of=/swapfile bs=1M count=1024 chmod 600 /swapfile mkswap /swapfile swapon /swapfile /swapfile swap swap defaults 0 0

LVM

pv / vg / lv / extend / resize
Physical Volumes
pvcreate /dev/sdb1 /dev/sdc1 pvs pvdisplay pvdisplay /dev/sdb1 pvremove /dev/sdb1
Volume Groups
vgcreate vgdata /dev/sdb1 vgs vgdisplay vgdata vgextend vgdata /dev/sdc1 vgreduce vgdata /dev/sdc1 vgremove vgdata
Logical Volumes
lvcreate -L 10G -n lvdata vgdata lvcreate -l 100%FREE -n lvdata vgdata lvcreate -l 50%VG -n lvdata vgdata lvs lvdisplay /dev/vgdata/lvdata lvremove /dev/vgdata/lvdata
Format & Mount LV
mkfs.xfs /dev/vgdata/lvdata mkdir /data mount /dev/vgdata/lvdata /data /dev/vgdata/lvdata /data xfs defaults 0 0
Extend LV (XFS — online)
lvextend -L +5G /dev/vgdata/lvdata xfs_growfs /data lvextend -L +5G -r /dev/vgdata/lvdata
-r flag with lvextend auto-resizes the filesystem. Works for both XFS and ext4.
Extend LV (ext4)
lvextend -L +5G /dev/vgdata/lvdata resize2fs /dev/vgdata/lvdata
Reduce LV (ext4 only, offline)
umount /data e2fsck -f /dev/vgdata/lvdata resize2fs /dev/vgdata/lvdata 8G lvreduce -L 8G /dev/vgdata/lvdata
XFS cannot be shrunk. Only ext4 supports lvreduce. Always e2fsck before reducing.

SELinux

contexts / booleans / audit / semanage
Modes
getenforce setenforce 0 setenforce 1
enforcingblocks & logs violations
permissivelogs only, no block
disabledcompletely off (reboot req.)
Set permanent mode in /etc/selinux/config — SELINUX=enforcing/permissive/disabled. Requires reboot.
View Contexts
ls -Z /var/www/html/ ps -Z ps -eZ | grep httpd id -Z
Change Context (Temporary)
chcon -t httpd_sys_content_t /var/www/html/file.html chcon -R -t httpd_sys_content_t /var/www/html/
chcon is NOT persistent. A relabel (restorecon or /.autorelabel) reverts it.
Restore Context
restorecon -Rv /var/www/html/ restorecon -v /var/www/html/file.html
Permanent Context (semanage)
semanage fcontext -a -t httpd_sys_content_t '/mydir(/.*)?' restorecon -Rv /mydir semanage fcontext -l semanage fcontext -l | grep /var/www
Pattern: semanage fcontext -a -t TYPE 'PATH(/.*)?' then restorecon -Rv PATH
Booleans
getsebool -a | grep httpd setsebool -P httpd_can_network_connect on setsebool -P httpd_enable_homedirs on setsebool -P ftpd_anon_write on semanage boolean -l | grep httpd
Troubleshoot
ausearch -m avc -ts recent ausearch -m avc -ts today audit2why < /var/log/audit/audit.log audit2allow -a cat /var/log/audit/audit.log | grep denied

Firewall (firewalld)

zones / services / ports / rules
Status & Zones
firewall-cmd --state firewall-cmd --list-all firewall-cmd --get-active-zones firewall-cmd --get-default-zone firewall-cmd --list-all-zones
Services
firewall-cmd --add-service=http --permanent firewall-cmd --add-service=https --permanent firewall-cmd --add-service=ssh --permanent firewall-cmd --remove-service=http --permanent firewall-cmd --list-services firewall-cmd --get-services
Ports
firewall-cmd --add-port=8080/tcp --permanent firewall-cmd --add-port=3306/tcp --permanent firewall-cmd --remove-port=8080/tcp --permanent firewall-cmd --list-ports
Source / Rich Rules
firewall-cmd --zone=public --add-source=192.168.1.0/24 --permanent firewall-cmd --zone=trusted --add-source=10.0.0.0/8 --permanent
Apply & Persist
firewall-cmd --reload firewall-cmd --runtime-to-permanent systemctl enable --now firewalld
CRITICAL: --permanent writes config but does NOT activate immediately. Always run --reload after --permanent changes, or use both: once without --permanent (instant) and once with --permanent (persisted).
Zone Management
firewall-cmd --new-zone=myzone --permanent firewall-cmd --zone=myzone --add-interface=eth1 --permanent firewall-cmd --set-default-zone=trusted

poweroff.targetrunlevel 0
rescue.targetrunlevel 1 / single user
multi-user.targetrunlevel 3 / no GUI
graphical.targetrunlevel 5 / with GUI
reboot.targetrunlevel 6
Target Commands

GRUB2
grub2-mkconfig -o /boot/grub2/grub.cfg vi /etc/default/grub GRUB_TIMEOUT=5 GRUB_CMDLINE_LINUX="..."
On EFI: grub2-mkconfig -o /boot/efi/EFI/redhat/grub.cfg
Root Password Reset (rd.break)
Step 1Reboot, press e at GRUB menu
Step 2Find linux line, append rd.break
Step 3Ctrl+X to boot
mount -o remount,rw /sysroot chroot /sysroot passwd root touch /.autorelabel exit Ctrl+D (or exit again)
/.autorelabel is critical — without it, SELinux context on /etc/shadow will be wrong and root login will fail after reboot.
Emergency / Rescue Mode (from GRUB)
systemd.unit=rescue.targetrescue shell
systemd.unit=emergency.targetminimal emergency
Append to end of linux line at GRUB. Ctrl+X to boot with modification.

NFS & AutoFS

exports / client mount / automount
NFS Server Setup
dnf install nfs-utils -y vi /etc/exports /shared 192.168.1.0/24(rw,sync,no_root_squash) /shared *(ro,sync) /data server2.example.com(rw,sync) exportfs -ra exportfs -v systemctl enable --now nfs-server firewall-cmd --add-service=nfs --permanent firewall-cmd --add-service=rpc-bind --permanent firewall-cmd --add-service=mountd --permanent firewall-cmd --reload
NFS Export Options
rwread/write access
roread only
syncsynchronous writes
no_root_squashroot client = root server
root_squashroot mapped to nobody (default)
NFS Client
showmount -e server.example.com mkdir /mnt/nfsshare mount -t nfs server:/shared /mnt/nfsshare mount -t nfs -o ro server:/shared /mnt # /etc/fstab persistent: server:/shared /mnt/nfsshare nfs defaults 0 0 server:/shared /mnt nfs rw,soft,timeo=30 0 0
AutoFS Setup
dnf install autofs -y vi /etc/auto.master /mnt /etc/auto.nfs vi /etc/auto.nfs share -rw,sync server:/shared * -rw,sync server:/home/& systemctl enable --now autofs ls /mnt/share
AutoFS mounts on-demand and unmounts after idle timeout. The * wildcard in auto.nfs maps /mnt/username to server:/home/username — used for home directory automounting.
Direct maps: /- /etc/auto.direct — maps specific absolute paths instead of relative names under a parent directory.
NFS + Firewall
NFS server needs nfs, rpc-bind, and mountd services in firewalld. All three — missing one breaks mounts.
Podman Rootless
Rootless containers use ~/.config/systemd/user/. Use systemctl --user commands. loginctl enable-linger for persistence without login.
Verify Everything
After each task: systemctl status, firewall-cmd --list-all, ls -Z, getfacl. Confirm, don't assume.
fstab Safety
After editing /etc/fstab, always run mount -a to test. A broken fstab entry can prevent the system from booting.
