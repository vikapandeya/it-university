User & Group Management — Multi-User Environment Setup
Beginner⏱️ 30 minUbuntu/RHEL
▼

Objective

Set up a multi-user environment with proper group memberships, password policies, and home directory permissions.

Steps

# Create groups
sudo groupadd developers
sudo groupadd devops
sudo groupadd readonly

# Create users with home directories
sudo useradd -m -s /bin/bash -G developers alice
sudo useradd -m -s /bin/bash -G developers,devops bob
sudo useradd -m -s /bin/bash -G readonly charlie

# Set passwords
echo "alice:SecurePass123!" | sudo chpasswd
echo "bob:SecurePass123!" | sudo chpasswd

# Force password change on first login
sudo chage -d 0 alice
sudo chage -d 0 bob

# Set password expiry (90 days)
sudo chage -M 90 alice
sudo chage -l alice  # verify

# Restrict charlie — no shell (service account)
sudo usermod -s /sbin/nologin charlie

# Lock an account
sudo usermod -L charlie
sudo passwd -l charlie  # alternative

# View group membership
id alice
groups bob

Verify

grep -E 'alice|bob|charlie' /etc/passwd
grep -E 'developers|devops|readonly' /etc/group
su - alice  # should prompt for password change

Linux Permissions, SUID/SGID, and ACLs
Beginner⏱️ 30 min
▼

Objective

Understand Linux permission bits, special permissions, and ACLs for fine-grained file access control.

# Basic permissions
mkdir /opt/project
chmod 2775 /opt/project   # SGID: new files inherit group
chown root:developers /opt/project

# Test shared directory
su - alice
touch /opt/project/alice.txt
ls -la /opt/project/  # group should be 'developers'

# ACLs — give bob read-only on alice's home
setfacl -m u:bob:r-- /home/alice
getfacl /home/alice

# Give developers group rwx on specific directory
setfacl -m g:developers:rwx /opt/project
setfacl -m d:g:developers:rwx /opt/project  # default ACL for new files

# SUID example (understand why it's dangerous)
ls -la /usr/bin/passwd  # Note the 's' bit — runs as root

# Find all SUID binaries (security audit)
find / -perm -4000 -type f 2>/dev/null

# Immutable file (cannot delete even as root)
touch /etc/important.conf
chattr +i /etc/important.conf
rm /etc/important.conf       # Operation not permitted
chattr -i /etc/important.conf  # remove immutable flag

LVM — Create, Extend, and Snapshot Logical Volumes
Intermediate⏱️ 45 min
▼

Objective

Set up LVM on a spare disk, create logical volumes, extend live volumes, and take snapshots for backup.

⚠️ Use a spare disk (/dev/sdb). Never run pvcreate on your OS disk.
# Initialize physical volume
sudo pvcreate /dev/sdb
pvdisplay /dev/sdb

# Create volume group
sudo vgcreate datavg /dev/sdb
vgdisplay datavg

# Create logical volumes
sudo lvcreate -L 5G -n datalv datavg
sudo lvcreate -L 2G -n loglv datavg

# Format and mount
sudo mkfs.ext4 /dev/datavg/datalv
sudo mkdir -p /mnt/data
sudo mount /dev/datavg/datalv /mnt/data

# Persistent mount in fstab
echo '/dev/datavg/datalv /mnt/data ext4 defaults 0 2' | sudo tee -a /etc/fstab

# Extend volume while mounted (online resize)
sudo lvextend -L +3G /dev/datavg/datalv
sudo resize2fs /dev/datavg/datalv  # resize filesystem
df -h /mnt/data  # should show new size

# Take snapshot for backup
sudo lvcreate -L 1G -s -n data-snap /dev/datavg/datalv
lvs  # see snapshot status

# Mount snapshot read-only for backup
sudo mount -o ro /dev/datavg/data-snap /mnt/snap-backup
rsync -av /mnt/snap-backup/ /backup/

# Remove snapshot
sudo umount /mnt/snap-backup
sudo lvremove /dev/datavg/data-snap

Bash Scripting — Automated Deployment Script with Error Handling
Intermediate⏱️ 60 min
▼

Objective

Write a production-grade bash deployment script with error handling, logging, and rollback capability.

#!/bin/bash
set -euo pipefail
IFS=$'\n\t'

# Config
APP_NAME="myapp"
DEPLOY_DIR="/opt/${APP_NAME}"
BACKUP_DIR="/opt/${APP_NAME}-backups"
LOG_FILE="/var/log/${APP_NAME}-deploy.log"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

# Logging
log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" | tee -a "$LOG_FILE"; }
error() { log "ERROR: $*" >&2; exit 1; }

# Rollback function
rollback() {
  log "Rolling back to previous version..."
  local latest_backup=$(ls -t "${BACKUP_DIR}"/*.tar.gz 2>/dev/null | head -1)
  [[ -z "$latest_backup" ]] && error "No backup found for rollback"
  tar -xzf "$latest_backup" -C "$(dirname "$DEPLOY_DIR")"
  systemctl restart "$APP_NAME"
  log "Rollback complete from: $latest_backup"
}
trap rollback ERR  # auto-rollback on any error

# Pre-flight checks
[[ $EUID -eq 0 ]] || error "Must run as root"
command -v systemctl &>/dev/null || error "systemctl not found"

# Backup current deployment
log "Creating backup..."
mkdir -p "$BACKUP_DIR"
[[ -d "$DEPLOY_DIR" ]] && tar -czf "${BACKUP_DIR}/${APP_NAME}-${TIMESTAMP}.tar.gz" "$DEPLOY_DIR"

# Deploy (simulated)
log "Deploying ${APP_NAME}..."
mkdir -p "$DEPLOY_DIR"
cp -r /tmp/new-build/* "$DEPLOY_DIR/"
chown -R www-data:www-data "$DEPLOY_DIR"

# Restart and health check
log "Restarting service..."
systemctl restart "$APP_NAME"
sleep 5

# Health check with retry
for i in {1..5}; do
  if curl -sf http://localhost:8080/health; then
    log "Health check passed on attempt $i"
    break
  fi
  [[ $i -eq 5 ]] && error "Health check failed after 5 attempts"
  log "Health check failed, retrying ($i/5)..."
  sleep 10
done

log "Deployment complete: ${APP_NAME} v${TIMESTAMP}"

SSH Hardening & Key-Based Authentication Setup
Intermediate⏱️ 30 min
▼

Objective

Configure SSH server for key-based auth only with security hardening. Set up SSH config on client.

⚠️ Always test config in a second terminal before closing existing SSH session.
# Client: Generate Ed25519 key (stronger than RSA)
ssh-keygen -t ed25519 -C "admin@myserver" -f ~/.ssh/myserver_ed25519

# Copy public key to server
ssh-copy-id -i ~/.ssh/myserver_ed25519.pub admin@192.168.1.100
# Or manually: cat ~/.ssh/myserver_ed25519.pub >> ~/.ssh/authorized_keys on server

# Server: Harden /etc/ssh/sshd_config
sudo vim /etc/ssh/sshd_config
# /etc/ssh/sshd_config — secure settings
Port 2222                        # non-standard port
AddressFamily inet               # IPv4 only
ListenAddress 192.168.1.100      # bind to specific interface
PermitRootLogin no               # never allow root login
PasswordAuthentication no        # keys only
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
MaxAuthTries 3
LoginGraceTime 30
ClientAliveInterval 300
ClientAliveCountMax 2
AllowUsers admin devops          # whitelist users
X11Forwarding no
AllowTcpForwarding no
Banner /etc/issue.net
# Reload SSH (don't restart — keeps current sessions)
sudo sshd -t  # test config first!
sudo systemctl reload sshd

# Client ~/.ssh/config
Host myserver
  HostName 192.168.1.100
  Port 2222
  User admin
  IdentityFile ~/.ssh/myserver_ed25519
  ServerAliveInterval 60

# Now connect simply:
ssh myserver

▼

Objective

Create and manage a custom systemd service unit, configure socket activation, and set up a timer unit.

# Simple service unit
sudo vim /etc/systemd/system/myapp.service
[Unit]
Description=My Application Server
Documentation=https://myapp.example.com/docs
After=network.target postgresql.service
Requires=network.target

[Service]
Type=simple
User=myapp
Group=myapp
WorkingDirectory=/opt/myapp
ExecStartPre=/opt/myapp/scripts/check-db.sh
ExecStart=/opt/myapp/bin/server --port 8080 --config /etc/myapp/config.yaml
ExecReload=/bin/kill -HUP $MAINPID
ExecStop=/bin/kill -TERM $MAINPID
TimeoutStopSec=30
Restart=on-failure
RestartSec=5s
StartLimitInterval=60s
StartLimitBurst=3
StandardOutput=journal
StandardError=journal
SyslogIdentifier=myapp
Environment="LOG_LEVEL=info"
EnvironmentFile=-/etc/myapp/env
LimitNOFILE=65536
PrivateTmp=true
NoNewPrivileges=true

[Install]
WantedBy=multi-user.target
# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable --now myapp.service
sudo systemctl status myapp

# Timer unit (run cleanup daily at 2AM)
sudo vim /etc/systemd/system/myapp-cleanup.timer
[Unit]
Description=Daily cleanup for myapp

[Timer]
OnCalendar=*-*-* 02:00:00
Persistent=true
Unit=myapp-cleanup.service

[Install]
WantedBy=timers.target
# Logs
journalctl -u myapp -f               # follow
journalctl -u myapp --since "1 hour ago"
journalctl -u myapp -p err           # errors only

Nginx Reverse Proxy with SSL Termination and Rate Limiting
Intermediate⏱️ 60 min
▼
sudo apt install nginx certbot python3-certbot-nginx

# /etc/nginx/sites-available/myapp.conf
limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;

upstream backend {
    server 127.0.0.1:8080;
    server 127.0.0.1:8081;
    keepalive 32;
}

server {
    listen 80;
    server_name api.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.example.com;

    ssl_certificate /etc/letsencrypt/live/api.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_session_cache shared:SSL:10m;
    add_header Strict-Transport-Security "max-age=31536000" always;

    location /api/ {
        limit_req zone=api burst=20 nodelay;
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }
}

# Get SSL cert (Let's Encrypt)
sudo certbot --nginx -d api.example.com
sudo nginx -t && sudo systemctl reload nginx

SELinux Troubleshooting — Custom Policy for Web App
Advanced⏱️ 60 minRHEL/CentOS
▼
# Check SELinux status
getenforce
sestatus

# Put in permissive to capture denials without blocking
setenforce 0

# Run your app and generate denials, then look at audit log
ausearch -m avc -ts recent | audit2why

# Example: nginx serving from non-standard dir
mkdir /data/webapp
# nginx can't access /data because context is wrong
ls -Z /data/webapp   # shows unlabeled_t

# Fix context
semanage fcontext -a -t httpd_sys_content_t "/data/webapp(/.*)?"
restorecon -Rv /data/webapp
ls -Z /data/webapp   # now shows httpd_sys_content_t

# If nginx needs to connect to network (e.g., proxy to backend)
getsebool -a | grep httpd_can_network
setsebool -P httpd_can_network_connect on

# Allow nginx on custom port 8443
semanage port -a -t http_port_t -p tcp 8443
semanage port -l | grep http_port_t

# Generate custom policy from audit denials
ausearch -m avc -ts recent | audit2allow -M mypolicy
semodule -i mypolicy.pp

# Return to enforcing
setenforce 1

L11 — iptables/nftables Firewall Rules

# Allow established + loopback
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -A INPUT -i lo -j ACCEPT
# Allow SSH on port 2222
iptables -A INPUT -p tcp --dport 2222 -j ACCEPT
# Allow HTTP/S
iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT
# Drop everything else
iptables -P INPUT DROP
# Rate limit SSH (anti-brute force)
iptables -I INPUT -p tcp --dport 2222 -m recent --set --name SSH
iptables -I INPUT -p tcp --dport 2222 -m recent --update --seconds 60 --hitcount 5 --name SSH -j DROP
# Save
iptables-save > /etc/iptables/rules.v4

Multi-Stage Build — Minimal Production Python Image
Beginner⏱️ 30 min
▼
# Dockerfile.multistage
# Stage 1: Build/install dependencies
FROM python:3.12-slim AS builder
WORKDIR /app
COPY requirements.txt .
RUN pip install --user --no-cache-dir -r requirements.txt

# Stage 2: Minimal runtime image
FROM python:3.12-slim AS runtime
# Create non-root user
RUN groupadd -r appuser && useradd -r -g appuser appuser
WORKDIR /app
# Copy only installed packages from builder
COPY --from=builder /root/.local /root/.local
COPY --chown=appuser:appuser src/ .
USER appuser
ENV PATH=/root/.local/bin:$PATH
EXPOSE 8000
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1
ENTRYPOINT ["python", "-m", "uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

# Build and compare sizes
docker build -t myapp:latest .
docker build --target builder -t myapp:builder .
docker images | grep myapp  # runtime should be MUCH smaller

# Scan for vulnerabilities
docker run --rm -v /var/run/docker.sock:/var/run/docker.sock \
    aquasec/trivy:latest image myapp:latest

      db:
        condition: service_healthy

      - backend

    volumes:
      - redisdata:/data
    networks:
      - backend

volumes:
  pgdata:
  redisdata:

networks:
  frontend:
  backend:

import anthropic
import chromadb
from chromadb.utils import embedding_functions
from pathlib import Path

# Setup
client = anthropic.Anthropic()  # uses ANTHROPIC_API_KEY
chroma_client = chromadb.PersistentClient(path="./chroma_db")
ef = embedding_functions.SentenceTransformerEmbeddingFunction(
    model_name="all-MiniLM-L6-v2"
)

# Create collection
collection = chroma_client.get_or_create_collection(
    name="docs",
    embedding_function=ef,
    metadata={"hnsw:space": "cosine"}
)

# Ingest documents with chunking
def chunk_text(text: str, chunk_size: int = 500, overlap: int = 50):
    words = text.split()
    chunks = []
    for i in range(0, len(words), chunk_size - overlap):
        chunk = " ".join(words[i:i + chunk_size])
        if chunk:
            chunks.append(chunk)
    return chunks

def ingest_file(filepath: str):
    text = Path(filepath).read_text()
    chunks = chunk_text(text)
    collection.upsert(
        documents=chunks,
        ids=[f"{filepath}:{i}" for i in range(len(chunks))],
        metadatas=[{"source": filepath, "chunk": i} for i in range(len(chunks))]
    )
    print(f"Ingested {len(chunks)} chunks from {filepath}")

# Query function
def rag_query(question: str, k: int = 3) -> str:
    # Retrieve relevant chunks
    results = collection.query(
        query_texts=[question],
        n_results=k,
        include=["documents", "distances", "metadatas"]
    )
    context = "\n\n".join(results["documents"][0])
    sources = [m["source"] for m in results["metadatas"][0]]

    # Generate with Claude
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        system=f"""You are a helpful assistant. Answer questions based ONLY on the provided context.
If the answer is not in the context, say "I don't have information about that."

Context:
{context}""",
        messages=[{"role": "user", "content": question}]
    )
    return {
        "answer": response.content[0].text,
        "sources": list(set(sources)),
        "chunks_used": len(results["documents"][0])
    }

# Usage
ingest_file("my-docs.txt")
result = rag_query("How does authentication work?")
print(result["answer"])
print("Sources:", result["sources"])

import anthropic, json, subprocess
from pathlib import Path

client = anthropic.Anthropic()

# Define tools
tools = [
    {
        "name": "read_file",
        "description": "Read the contents of a file",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string", "description": "File path to read"}
            },
            "required": ["path"]
        }
    },
    {
        "name": "run_command",
        "description": "Run a shell command and return output. Only for safe commands.",
        "input_schema": {
            "type": "object",
            "properties": {
                "command": {"type": "string", "description": "The shell command"},
                "timeout": {"type": "integer", "default": 30}
            },
            "required": ["command"]
        }
    }
]

# Tool execution
def execute_tool(name: str, inputs: dict) -> str:
    if name == "read_file":
        try:
            return Path(inputs["path"]).read_text()
        except Exception as e:
            return f"Error reading file: {e}"
    elif name == "run_command":
        # SAFETY: In production, whitelist allowed commands!
        result = subprocess.run(
            inputs["command"], shell=True, capture_output=True,
            text=True, timeout=inputs.get("timeout", 30)
        )
        return result.stdout + result.stderr
    return "Unknown tool"

# Agentic loop
def agent(task: str) -> str:
    messages = [{"role": "user", "content": task}]
    while True:
        response = client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=4096,
            tools=tools,
            messages=messages
        )
        # Check if done
        if response.stop_reason == "end_turn":
            return next(b.text for b in response.content if hasattr(b, "text"))
        # Process tool calls
        messages.append({"role": "assistant", "content": response.content})
        tool_results = []
        for block in response.content:
            if block.type == "tool_use":
                result = execute_tool(block.name, block.input)
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": result
                })
        messages.append({"role": "user", "content": tool_results})

# Run
print(agent("Read the file README.md and tell me what this project does, then run 'git log --oneline -5' to show recent commits."))
