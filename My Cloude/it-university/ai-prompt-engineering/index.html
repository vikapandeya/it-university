RLHF — How ChatGPT was trained

# Reinforcement Learning from Human Feedback (RLHF)
# Step 1: Supervised Fine-Tuning (SFT)
#   — Start from pre-trained GPT
#   — Fine-tune on human-written demonstrations (prompt → ideal response)

# Step 2: Reward Model Training
#   — Generate multiple responses to prompts
#   — Human rankers rank responses (best to worst)
#   — Train reward model to predict human preference scores

# Step 3: PPO RL Training
#   — Use PPO (Proximal Policy Optimization) algorithm
#   — LLM generates response → reward model scores it
#   — PPO updates LLM to maximize reward score
#   — KL penalty prevents diverging too far from SFT model

# Constitutional AI (Anthropic/Claude alternative)
# Step 1: SFT on helpful responses
# Step 2: Generate harmful response → have model critique it → rewrite
# Step 3: RL against AI preference model (not human ranking)
# — Scales better, more consistent principles

⚡ Transformer Architecture

Self-Attention, Encoder vs Decoder, Scaling Laws

Self-Attention Mechanism

# "Attention is All You Need" (Vaswani et al., 2017)
# For each token, compute how much to attend to every other token

# Input embedding → 3 linear projections
# Q (Query) = X × Wq    — what am I looking for?
# K (Key)   = X × Wk    — what do I contain?
# V (Value) = X × Wv    — what do I contribute?

# Attention scores (scaled dot-product)
# Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) × V
# sqrt(d_k) = scale factor to prevent vanishing gradients

# Multi-Head Attention
# Run h attention heads in parallel (each with different W_q, W_k, W_v)
# Concatenate heads → linear projection
# Different heads learn different relationships:
#   Head 1: nearby tokens (syntax)
#   Head 2: coreference resolution
#   Head 3: semantic similarity

import torch, torch.nn.functional as F

def attention(Q, K, V, mask=None):
    d_k = Q.shape[-1]
    scores = Q @ K.transpose(-2, -1) / d_k ** 0.5  # scaled dot product
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)  # causal mask for decoder
    weights = F.softmax(scores, dim=-1)
    return weights @ V

Scaling Laws (Chinchilla)

# Kaplan et al. (OpenAI, 2020):
# Loss ~ N^(-0.076) × D^(-0.095) × C^(-0.050)
# N = model parameters, D = training tokens, C = compute

# Chinchilla (Hoffmann et al., DeepMind, 2022):
# Optimal: D ≈ 20 × N  (20 tokens per parameter)
# GPT-3 (175B params) was UNDERTRAINED
# Chinchilla (70B params, 1.4T tokens) > Gopher (280B, 300B tokens)

# Examples
# Llama 2 70B — trained on 2T tokens (optimal: ~1.4T)
# Llama 3 8B  — trained on 15T tokens (much more than optimal)
# More tokens > more parameters (up to a point)

# Context window evolution
# GPT-3:         4K tokens
# GPT-4:        32K tokens (turbo: 128K)
# Claude 3:    200K tokens
# Gemini 1.5:    1M tokens (research: 10M)
# Key techniques: RoPE, ALiBi, YaRN for long context

# Byte Pair Encoding (BPE) — most common tokenizer algorithm
# 1. Start with character vocabulary
# 2. Count most frequent adjacent pairs
# 3. Merge most frequent pair into new token
# 4. Repeat until vocabulary size reached (e.g., 50,000 tokens)

# Rule of thumb: ~1 token = ~4 chars (English)
# Code is more tokens-dense than English prose
# Non-Latin scripts (Chinese, Arabic) often 1 char = 1+ tokens

Zero-shot, Few-shot, Chain-of-Thought, ReAct

Zero-Shot vs Few-Shot

# Zero-Shot: just the task, no examples
"""
Classify the sentiment of this review as Positive, Negative, or Neutral.

Review: "The hotel was clean but the breakfast was disappointing."
Sentiment:
"""

# Few-Shot: provide examples to guide format and reasoning
"""
Classify the sentiment:

Review: "Amazing experience, would definitely return!" → Positive
Review: "Terrible service, waited 2 hours." → Negative
Review: "Room was okay, nothing special." → Neutral

Review: "The hotel was clean but the breakfast was disappointing."
Sentiment:
"""

Chain-of-Thought (CoT)

# Standard prompt (fails on multi-step reasoning)
"""Q: Roger has 5 tennis balls. He buys 2 cans of balls, 3 balls each. How many?"""
# Model may answer: 8 (wrong) or 11 (correct)

# Chain-of-Thought: "Let's think step by step"
"""
Q: Roger has 5 tennis balls. He buys 2 cans of 3 balls each. How many balls does he have now?

Let's think step by step:
- Roger starts with 5 balls
- He buys 2 cans × 3 balls = 6 balls
- Total: 5 + 6 = 11 balls

A: 11 balls
"""

# Zero-shot CoT magic phrase
"Let's think step by step."
# Improves accuracy on math/logic by 20-40%

ReAct (Reason + Act)

# ReAct: interleave reasoning and tool use
# Pattern: Thought → Action → Observation → Thought → ...

"""
Question: What is the current CEO of the company that makes Claude?

Thought: I need to find who makes Claude, then find their CEO.
Action: Search["Claude AI maker"]
Observation: Claude is made by Anthropic.
Thought: Now I need to find Anthropic's CEO.
Action: Search["Anthropic CEO"]
Observation: Dario Amodei is the CEO of Anthropic.
Thought: I have the answer.
Answer: Dario Amodei is the CEO of Anthropic, the company that makes Claude.
"""

Tree of Thoughts (ToT)

# ToT: explore multiple reasoning paths simultaneously
# Like a game tree — evaluate intermediate thoughts
# Best for: math problems, planning, creative writing

# 1. Generate multiple thought candidates (branches)
# 2. Evaluate each thought (good/bad/unsure)
# 3. Explore promising branches further
# 4. Backtrack if stuck
# 5. Return best solution found

# Self-Consistency: sample multiple CoT paths, take majority vote
# Works well for math: 40 independent paths → vote on final answer

DSPy — Systematic Prompt Optimization

import dspy

# DSPy: program your prompts instead of hand-crafting them
# Automatically optimize prompts using a development set

# Define a signature (input → output types)
class SentimentAnalyzer(dspy.Signature):
    """Analyze the sentiment of a product review."""
    review = dspy.InputField()
    sentiment = dspy.OutputField(desc="Positive, Negative, or Neutral")

# Create a module
analyzer = dspy.Predict(SentimentAnalyzer)

# Optimize with examples (BootstrapFewShot, etc.)
from dspy.teleprompt import BootstrapFewShot

trainset = [
    dspy.Example(review="Great product!", sentiment="Positive"),
    dspy.Example(review="Waste of money", sentiment="Negative")
]

teleprompter = BootstrapFewShot(metric=lambda x, y: x.sentiment == y.sentiment)
optimized = teleprompter.compile(analyzer, trainset=trainset)

Tool Use (Function Calling)

import anthropic, json

client = anthropic.Anthropic()

tools = [{
    "name": "get_weather",
    "description": "Get current weather for a location",
    "input_schema": {
        "type": "object",
        "properties": {
            "location": {"type": "string", "description": "City name"},
            "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
        },
        "required": ["location"]
    }
}]

def get_weather(location, unit="celsius"):
    # Real implementation would call weather API
    return {"location": location, "temperature": 22, "condition": "Sunny", "unit": unit}

# Initial message
messages = [{"role": "user", "content": "What's the weather in London?"}]

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    tools=tools,
    messages=messages
)

# Handle tool call
if response.stop_reason == "tool_use":
    tool_use = next(b for b in response.content if b.type == "tool_use")
    tool_result = get_weather(**tool_use.input)

    messages.extend([
        {"role": "assistant", "content": response.content},
        {"role": "user", "content": [{
            "type": "tool_result",
            "tool_use_id": tool_use.id,
            "content": json.dumps(tool_result)
        }]}
    ])

    final = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        tools=tools,
        messages=messages
    )
    print(final.content[0].text)

# Chunking strategies
def chunk_by_tokens(text, chunk_size=512, overlap=50):
    import tiktoken
    enc = tiktoken.get_encoding("cl100k_base")
    tokens = enc.encode(text)
    chunks = []
    for i in range(0, len(tokens), chunk_size - overlap):
        chunk = enc.decode(tokens[i:i+chunk_size])
        chunks.append(chunk)
    return chunks

# Semantic chunking (split at natural boundaries)

from chromadb.utils import embedding_functions

def rag_query(question: str, n_results: int = 3) -> str:
    results = collection.query(query_texts=[question], n_results=n_results)

        messages=[{
            "role": "user",

        }]
    )
    return response.content[0].text

Advanced Retrieval Techniques

# HyDE — Hypothetical Document Embeddings
# Generate a hypothetical answer → embed it → search
# Often finds better matches than embedding the question directly

# Multi-query retrieval
# Generate 3 variations of the question → search with each → dedupe
from langchain.retrievers.multi_query import MultiQueryRetriever

# Parent-Child chunking
# Index small chunks for retrieval, return large parent chunk as context
# Keeps context coherent, improves retrieval precision

# Reranking
from sentence_transformers import CrossEncoder
reranker = CrossEncoder('cross-encoder/ms-marco-MiniLM-L-6-v2')

def rerank(query, documents, top_k=3):
    pairs = [[query, doc] for doc in documents]
    scores = reranker.predict(pairs)
    ranked = sorted(zip(scores, documents), reverse=True)
    return [doc for _, doc in ranked[:top_k]]

from typing import TypedDict, Annotated
import operator

class AgentState(TypedDict):
    messages: Annotated[list, operator.add]
    next: str

def researcher(state: AgentState):
    # Search the web, return findings
    ...
    return {"messages": [{"role": "assistant", "content": findings}]}

def writer(state: AgentState):
    # Write report based on research
    ...
    return {"messages": [{"role": "assistant", "content": report}]}

def router(state: AgentState):
    last_msg = state["messages"][-1]["content"]
    if "RESEARCH_DONE" in last_msg:
        return "writer"
    return "researcher"

workflow = StateGraph(AgentState)
workflow.add_node("researcher", researcher)
workflow.add_node("writer", writer)
workflow.add_edge("writer", END)
workflow.add_conditional_edges("researcher", router, {"researcher": "researcher", "writer": "writer"})
workflow.set_entry_point("researcher")

app = workflow.compile()
result = app.invoke({"messages": [{"role": "user", "content": "Research Claude Sonnet 4.6"}]})

Agents, Tasks, Tools & Crews
from crewai import Agent, Task, Crew, Process
from crewai_tools import SerperDevTool, WebsiteSearchTool

# Tools
search_tool = SerperDevTool()

# Agents
security_researcher = Agent(
    role="Senior Security Researcher",
    goal="Investigate and document the latest cybersecurity threats",
    backstory="Expert with 15 years in threat intelligence and incident response.",
    tools=[search_tool],
    llm="claude-sonnet-4-6",
    verbose=True,
    allow_delegation=False,
    max_iter=3
)

report_writer = Agent(
    role="Technical Report Writer",
    goal="Write comprehensive, clear security reports",
    backstory="Experienced technical writer who translates complex security concepts.",
    llm="claude-sonnet-4-6",
    verbose=True
)

# Tasks
research_task = Task(
    description="""
    Research the top 3 active ransomware groups in 2025.
    For each: TTPs, target industries, notable incidents.
    Use search tools to find current information.
    """,
    expected_output="Detailed research notes on 3 ransomware groups with TTPs and IOCs",
    agent=security_researcher
)

report_task = Task(
    description="Write an executive threat report based on the research.",
    expected_output="Professional 2-page executive report with recommendations",
    agent=report_writer,
    context=[research_task]    # depends on research_task output
)

# Crew
crew = Crew(
    agents=[security_researcher, report_writer],
    tasks=[research_task, report_task],
    process=Process.sequential,   # or Process.hierarchical (manager agent)
    verbose=True
)

result = crew.kickoff()
print(result.raw)

🔧 Fine-Tuning with LoRA/QLoRA

Parameter-Efficient Fine-Tuning (PEFT)
# LoRA — Low-Rank Adaptation
# Instead of fine-tuning all parameters (7B = 14GB GPU)
# Inject small trainable matrices (rank r) alongside frozen weights
# W' = W + B×A  where B (d×r), A (r×d), r << d
# Typical r=8 or r=16: train only 0.1% of parameters!

from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, TaskType, prepare_model_for_kbit_training
import torch

# QLoRA: 4-bit quantization + LoRA (train 7B on single 16GB GPU)
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4"
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-3.1-8B-Instruct",
    quantization_config=bnb_config,
    device_map="auto"
)
model = prepare_model_for_kbit_training(model)

lora_config = LoraConfig(
    r=16,                           # rank — higher = more params
    lora_alpha=32,                  # scaling factor
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.1,
    task_type=TaskType.CAUSAL_LM
)
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# trainable params: 13,107,200 || all params: 8,030,261,248 || 0.163%

# Training with SFTTrainer (TRL library)
from trl import SFTTrainer
from transformers import TrainingArguments

args = TrainingArguments(
    output_dir="./llama-finetuned",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    fp16=True,
    logging_steps=10,
    save_strategy="epoch",
    warmup_ratio=0.03,
    lr_scheduler_type="cosine"
)

trainer = SFTTrainer(
    model=model,
    args=args,
    train_dataset=dataset,
    dataset_text_field="text",
    max_seq_length=2048
)
trainer.train()
model.save_pretrained("./llama-finetuned")

🎯 AI Interview Questions

Top 15 AI/ML Interview Q&A

Q1: What is the difference between a parameter and a hyperparameter?

Parameter: Learned by the model during training (weights, biases). Model has billions of parameters.
Hyperparameter: Set before training, controls training process. Examples: learning rate, batch size, number of layers, dropout rate, rank (r) in LoRA.

Q2: What is hallucination in LLMs and how do you mitigate it?

Hallucination: LLM generates confident but factually incorrect information. Mitigation: RAG (ground responses in retrieved facts), temperature=0 (deterministic), chain-of-thought (forces step-by-step reasoning), citations (force model to cite sources), Constitutional AI, self-consistency (sample multiple paths, take majority vote).

Q3: What is the attention mechanism and why is it important?

Attention allows each token to look at all other tokens and determine how much to "attend" to them. Scales with O(n²) in sequence length. Key innovation: captures long-range dependencies that RNNs struggled with. Multi-head attention runs multiple attention patterns in parallel, each learning different relationships.

Q4: Explain RAG vs. Fine-tuning — when to use each?

RAG: Use when knowledge changes frequently (news, docs, products), when you need citations/sources, when data is too large to fit in context, or when you need to control what the model knows.
Fine-tuning: Use when you need a specific output format/style, domain-specific reasoning patterns, or to reduce prompt length. NOT for adding new facts (use RAG for that).

Q5: What is temperature and how does it affect output?

Temperature scales the logits before softmax. Low temperature (0.0-0.3): more deterministic, picks highest probability tokens. Good for code, facts. High temperature (0.7-1.0): more random, creative, varied. Good for creative writing. Temperature >1.0: can get incoherent. For factual Q&A: 0. For creative: 0.7-0.9.

Q6: What is the difference between tokens and embeddings?

Tokens: Discrete units (integers) — BPE subword pieces. "tokenization" → ["token", "ization"].
Embeddings: Dense vector representations (float arrays, e.g. 4096 dimensions) that capture semantic meaning. Similar words have similar embedding vectors. LLM maps tokens → embeddings as first step.

Q7: How does RLHF improve LLM behavior?

RLHF (Reinforcement Learning from Human Feedback): 1) Fine-tune on human demonstrations (SFT). 2) Collect pairwise human preferences (which response is better). 3) Train reward model on preferences. 4) Use PPO to optimize LLM to maximize reward while staying close to SFT model (KL constraint). Makes models more helpful, honest, and harmless.

Q8: What is a vector database and why is it needed for RAG?

Vector databases store high-dimensional embedding vectors and enable fast approximate nearest-neighbor (ANN) search using algorithms like HNSW or IVF. Needed for RAG because: 1) Text semantic similarity is captured in embedding space, not exact string matching. 2) Traditional SQL can't efficiently search 1536-dimensional vectors across millions of documents. Examples: Chroma, Pinecone, Qdrant, pgvector.
