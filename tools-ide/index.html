📖 Git Basics

What is Git? Core Workflow

Git is a free, distributed version control system created by Linus Torvalds in 2005. It tracks changes to files over time, lets you recall any version, collaborate with others, and work on features in isolation.

Git vs GitHub: Git is the tool (runs locally). GitHub is a cloud hosting service for Git repositories. GitLab and Bitbucket are alternatives.

Core Git Workflow

Working Directory
→ Files you edit locally
Staging Area
→ git add — snapshot ready to commit
Local Repo
→ git commit — saved to history
Remote Repo
→ git push — shared with team 
# First-time setup
git config --global user.name "Your Name"
git config --global user.email "you@email.com"
git config --global init.defaultBranch main

# Initialize new repo
git init                  # create .git folder
git add .                 # stage everything
git commit -m "Init"      # first commit

# Check state
git status                # what's changed?
git log --oneline         # commit history
git log --oneline --graph --all  # visual branch graph
git diff                  # unstaged changes
git diff --staged         # staged changes

Undoing Changes

git restore file.txt          # discard working dir changes
git restore --staged file.txt # unstage a file
git revert HEAD               # create undo commit (safe)
git reset --soft HEAD~1       # undo last commit, keep staged
git reset --hard HEAD~1       # undo last commit, DISCARD changes
git stash                     # save changes temporarily
git stash pop                 # restore stashed changes

🌿 Branching & Merging

Branches, Merge, Rebase

Branches let you work on features/fixes in isolation without affecting the main codebase.

# Create and switch to branch
git branch feature/login    # create
git checkout feature/login  # switch (classic)
git checkout -b feature/x   # create + switch
git switch -c feature/x     # modern syntax

# Merge feature into main
git checkout main
git merge feature/login     # merge
git branch -d feature/login # delete merged branch
git branch -D feature/login # force delete

# Rebase (cleaner linear history)
git rebase main             # replay commits on top of main
git cherry-pick abc1234     # pick one commit from another branch

# View branches
git branch                  # local branches
git branch -a               # all (including remote)
git branch -vv              # show tracking & last commit

Git Workflows

Git Flow ▾
Branches: main (production) · develop (integration) · feature/* (new features) · release/* (prepare release) · hotfix/* (urgent fixes). Complex but structured — great for versioned software with explicit release cycles.
GitHub Flow ▾
Simple: Create branch → commit changes → open Pull Request → review → merge to main → deploy. Used by GitHub, most web startups. Short-lived branches, continuous deployment. Best for web apps.
Trunk-Based Development ▾
All devs commit directly to main (or very short-lived feature branches <24h). Feature flags hide incomplete features. Requires strong CI/CD and test coverage. Fastest delivery cycle — used by Google, Facebook.

🔗 Remote & GitHub

Push, Pull, PRs, GitHub Actions
# Connect to remote
git remote add origin https://github.com/user/repo.git
git remote -v              # list remotes
git remote remove origin   # remove remote

# Push / Pull
git push -u origin main    # first push (sets upstream)
git push                   # subsequent pushes
git pull                   # fetch + merge (= git fetch + git merge)
git fetch                  # download but don't merge
git pull --rebase          # fetch + rebase (cleaner history)

# Clone existing repo
git clone https://github.com/user/repo.git
git clone https://github.com/user/repo.git my-folder

# Fork workflow (open source contribution)
git remote add upstream https://github.com/original/repo.git
git fetch upstream
git rebase upstream/main   # sync your fork

GitHub Features

  • Pull Requests (PR) — propose changes, request review, discuss, then merge. Core collaboration feature.
  • GitHub Actions — CI/CD pipelines in YAML. Auto-run tests, build, deploy on push/PR. Free for public repos.
  • Issues & Projects — bug tracking, feature requests, Kanban-style boards.
  • Dependabot — scans dependencies for CVEs, auto-creates PRs to patch vulnerabilities.
  • GitHub Pages — free static site hosting from any repo.
  • Codespaces — cloud-based VS Code environment for any repo.

GitHub Actions — Example Workflow

File Organization Best Practices

  • Projects folder — keep all dev projects under ~/Projects/ or C:\Dev\
  • Date-prefix backups — 2026-06-23_backup_project.zip
  • No spaces in paths — use hyphens or underscores in filenames
  • Meaningful names — invoice-2026-q2.pdf not document1.pdf
  • Version files — use Git instead of report_v2_FINAL_final.docx

📮 Postman & API Testing

API Testing, Collections & Automation

Core Concepts

ConceptWhat it is
CollectionGroup of saved API requests — organize by feature or service
EnvironmentVariables set (dev/staging/prod) — swap base URLs without editing requests
Pre-request ScriptJavaScript that runs BEFORE the request — generate tokens, set timestamps
TestsJavaScript assertions that run AFTER response — validate status, body, headers
RunnerRun entire collection sequentially, with data from CSV/JSON files
Mock ServerSimulate API responses before backend exists — unblock frontend dev

Writing Postman Tests

// Test: status code
pm.test("Status 200", () => {
  pm.response.to.have.status(200);
});

// Test: response body field
pm.test("User has ID", () => {
  const json = pm.response.json();
  pm.expect(json.id).to.be.a('number');
  pm.expect(json.email).to.include('@');
});

// Test: response time
pm.test("Response under 500ms", () => {
  pm.expect(pm.response.responseTime).to.be.below(500);
});

// Set variable from response for next request
const token = pm.response.json().access_token;
pm.environment.set("auth_token", token);

// Pre-request: generate HMAC signature
const crypto = require('crypto-js');
const timestamp = Date.now().toString();
const signature = crypto.HmacSHA256(timestamp, pm.environment.get("secret")).toString();
pm.request.headers.add({ key: 'X-Signature', value: signature });
pm.request.headers.add({ key: 'X-Timestamp', value: timestamp });

# Parse AWS CLI output
