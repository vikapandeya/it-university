Layers & Activations

Dense & Regularization
layers.Dense(64, activation='relu') layers.Dense(64, activation='relu', kernel_regularizer=tf.keras.regularizers.l2(0.01)) layers.Dropout(rate=0.3) # zeros out 30% of activations, training only layers.BatchNormalization() # normalize activations — add after Dense/Conv layers.Flatten() # convert multi-dim tensor to 1D
Activation Functions
relumax(0, x) — standard for hidden layers
sigmoid1/(1+e^-x) — binary output [0,1]
softmaxsum to 1 — multiclass output probabilities
tanh[-1,1] — recurrent layers, older models
linearf(x) = x — regression output layer
Output Layer by Task
Binary classificationDense(1, activation='sigmoid')
Multiclass (one-hot)Dense(n_classes, activation='softmax')
Multiclass (integer labels)Dense(n_classes, activation='softmax') + sparse_cat loss
Regression / time seriesDense(1) — linear activation (default)
Multi-output regressionDense(n_outputs) — linear
Common Layer Patterns
Dense→BN→ReLUBatchNorm before activation — modern pattern
Dense→ReLU→DropoutDropout after activation — classic pattern
BN in inferenceUses running mean/var — behavior differs from training
BatchNormalization: add after Dense/Conv2D, before activation. It stabilizes training and allows higher learning rates. For small models in the exam, it may not always help — test both.

CNN — Image Classification

Standard CNN Architecture
model = tf.keras.Sequential([   layers.Conv2D(32, (3,3), activation='relu', input_shape=(150,150,3)),   layers.MaxPooling2D(2,2),   layers.Conv2D(64, (3,3), activation='relu'),   layers.MaxPooling2D(2,2),   layers.Conv2D(128, (3,3), activation='relu'),   layers.MaxPooling2D(2,2),   layers.Flatten(),   layers.Dense(512, activation='relu'),   layers.Dense(1, activation='sigmoid') # or Dense(n, 'softmax') ])
Conv2D Key Arguments
filtersNumber of feature maps (32, 64, 128...)
kernel_size(3,3) standard — captures local patterns
padding='same'Output same size as input (zero-pad)
padding='valid'Default — shrinks spatial dimensions
stridesDefault (1,1) — controls downsampling
Pooling Alternatives
MaxPooling2D(2,2)Keep max value — retains strong features
AveragePooling2DAverage value — smoother, less common
GlobalAveragePooling2DMean per channel — replaces Flatten, no params
Loss & Output by Task
Binary (cats vs dogs)sigmoid output + binary_crossentropy
Multiclass (one-hot)softmax + categorical_crossentropy
Multiclass (integer)softmax + sparse_categorical_crossentropy
Rule of thumb: increase filters as you go deeper (32→64→128). MaxPooling halves spatial dims. GlobalAveragePooling2D is parameter-free — use it after last Conv block when you don't need Flatten+Dense.

Transfer Learning

Feature Extraction (base frozen)
base = tf.keras.applications.MobileNetV2(   input_shape=(224,224,3),   include_top=False,   weights='imagenet' ) base.trainable = False # freeze all base layers x = base.output x = layers.GlobalAveragePooling2D()(x) x = layers.Dense(128, activation='relu')(x) outputs = layers.Dense(1, activation='sigmoid')(x) model = tf.keras.Model(base.input, outputs)
Fine-Tuning (unfreeze top layers)
base.trainable = True # Freeze bottom layers, unfreeze top N for layer in base.layers[:-20]: # freeze all but last 20   layer.trainable = False # Recompile with LOW learning rate model.compile(optimizer=tf.keras.optimizers.Adam(1e-5), loss='binary_crossentropy', metrics=['accuracy']) model.fit(train_gen, epochs=10, ...) # train more epochs
Available Pretrained Models
MobileNetV2Lightweight — best for exam (fast to train)
InceptionV3input_shape=(299,299,3) — larger
VGG16Simple architecture, large model
ResNet50Residual connections, strong baseline
EfficientNetB0-B7State of art — scaling compound
When to Fine-tune vs Feature Extract
Small dataset + similar domainFeature extraction (frozen base)
Large dataset or different domainFine-tune (unfreeze top layers)
AlwaysStart frozen, then unfreeze top layers progressively
Critical: preprocess inputs for each model. MobileNetV2 expects tf.keras.applications.mobilenet_v2.preprocess_input (values in [-1,1]). VGG16: preprocess_input (caffe mode). Don't just divide by 255 for transfer learning.

Time Series & Sequences

Windowed Dataset Helper
def windowed_dataset(series, window_size, batch_size, shuffle_buffer):   dataset = tf.data.Dataset.from_tensor_slices(series)   dataset = dataset.window(window_size + 1, shift=1, drop_remainder=True)   dataset = dataset.flat_map(lambda w: w.batch(window_size + 1))   dataset = dataset.shuffle(shuffle_buffer)   dataset = dataset.map(lambda w: (w[:-1], w[-1:]))   return dataset.batch(batch_size).prefetch(1)

Overfitting Solutions
More dataBest solution — data augmentation for images
Dropoutrate=0.3–0.5 — regularize dense/LSTM layers
L2 regularizationkernel_regularizer=tf.keras.regularizers.l2()
Batch NormalizationRegularizing effect + faster convergence
Early StoppingStop before overfitting — restore_best_weights
Simpler modelReduce layers or units
Accuracy Not Improving
Check learning rateToo high → unstable. Too low → slow.
Check normalizationInput pixels should be [0,1] or [-1,1]
Check labelsVerify label encoding matches loss function
Check architectureModel may be too simple for the task
restore_best_weights=True in EarlyStopping restores the model to its best epoch when training stops. Without this, the model keeps the weights from the last (potentially worse) epoch.
