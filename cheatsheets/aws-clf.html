"Most cost-effective"Reserved > Savings Plans > Spot (if interruptible ok)
"Physical" / "hardware"AWS responsibility in shared model
"Application" / "data"Customer responsibility in shared model
"Compliance reports"AWS Artifact (not GuardDuty, not Config)
"Audit API calls"CloudTrail (not CloudWatch, not Config)
"Monitor metrics/logs"CloudWatch (not CloudTrail)
"Track config changes"AWS Config (not CloudTrail, not CloudWatch)
"Decouple components"SQS (pull) or SNS (push/pub-sub)
"CDN / edge caching"CloudFront (uses edge locations, NOT AZs)
"DNS / domain routing"Route 53 (only service with 100% SLA)
"Dedicated hardware"Dedicated Hosts (BYOL/compliance) or Dedicated Instances
"High availability DB"RDS Multi-AZ (standby replica, auto-failover)
"Scale DB reads"RDS Read Replica (async; can be cross-region)
"Global low latency"Global Accelerator or CloudFront
"Private line to AWS"Direct Connect (not VPN — VPN uses public internet)
"Batch / flexible jobs"Spot Instances — can tolerate interruption
"Sensitive data in S3"Amazon Macie (PII detection)
"Threat detection"Amazon GuardDuty (ML-based, no agents needed)
Common Traps
EC2 OS and application patching = CUSTOMER. Underlying hypervisor and physical hardware = AWS. RDS OS patching = AWS (fully managed).
Security Groups: STATEFUL (return traffic auto-allowed, rules are allow-only). NACLs: STATELESS (explicit rules for inbound AND outbound, can DENY).
CloudFront uses Edge Locations — NOT Availability Zones or Regions. Edge locations are separate from the Region AZ count.
Trusted Advisor full checks (all 5 categories) require Business support or higher. Basic/Developer only get 7 core checks (security & service limits).
AWS Artifact = compliance reports & agreements. It is NOT a security monitoring or threat detection service. Used for SOC, PCI, ISO documents.
Infrastructure Concepts
RegionGeographic area with 2+ AZs; independent fault domains
AZOne or more discrete data centers; physically separate
Edge LocationCloudFront PoP; caches content closer to users
Multi-AZDeploy across AZs = high availability (HA)
Multi-RegionDeploy across regions = disaster recovery (DR)
Health DashboardPersonal = YOUR alerts; Service = ALL AWS global status
Route 53 SLA100% availability SLA — the only AWS service with this
