Private vs Service Endpoints

PaaS connectivity security
Service Endpoints
Traffic pathAzure backbone, bypasses internet
Source IP to serviceVNet public IP (not private)
Private IP allocatedNo — no NIC in subnet
On-prem accessNOT accessible from on-prem VPN/ExpressRoute
DNS changeNo DNS override required
CostFree
Data exfiltrationNot prevented (any tenant's storage)
Private Endpoints
Traffic pathDedicated NIC with private IP in subnet
Source IP to servicePrivate IP — fully private
Private IP allocatedYES — NIC created in your subnet
On-prem accessYES — via VPN/ExpressRoute
DNS overrideprivatelink.blob.core.windows.net → private IP
Cost~$7.30/month per endpoint
Data exfiltrationPREVENTED — only your specific resource
When to Force Private Endpoint
CompliancePCI-DSS, HIPAA, ISO 27001 requirements
On-prem accessUsers on VPN/ExpressRoute need PaaS access
Data exfiltrationPrevent access to competitor storage accounts
Zero public exposureDisable public endpoint entirely
Private DNS Zones (Key)
Blob storageprivatelink.blob.core.windows.net
Key Vaultprivatelink.vaultcore.azure.net
SQL Databaseprivatelink.database.windows.net
ACRprivatelink.azurecr.io
Private Endpoints are strictly superior for security but cost more. Service Endpoints are fine for VNet-only workloads that don't need on-prem access or data exfiltration protection.

KQL Security Queries

Sentinel / Log Analytics
Failed Login Brute Force Detection
SecurityEvent | where EventID == 4625 | summarize FailedLogins=count() by Account, IpAddress | where FailedLogins > 10 | order by FailedLogins desc
Risky Sign-ins (High/Medium)
SigninLogs | where ResultType != 0 | where RiskLevelDuringSignIn in ("high","medium") | project TimeGenerated, UserPrincipalName, IPAddress, Location, RiskLevelDuringSignIn | order by TimeGenerated desc
Privileged Write/Delete Operations
AzureActivity | where OperationNameValue contains "write" or OperationNameValue contains "delete" | where ActivityStatusValue == "Success" | project TimeGenerated, Caller, OperationNameValue, ResourceGroup | order by TimeGenerated desc
Impossible Travel Detection
SigninLogs | where ResultType == 0 | extend City = tostring(LocationDetails.city) | extend Country = tostring(LocationDetails.countryOrRegion) | project TimeGenerated, UserPrincipalName, IPAddress, City, Country | join kind=inner ( SigninLogs | where ResultType == 0 | project TimeGenerated2=TimeGenerated, UserPrincipalName, Country2=tostring(LocationDetails.countryOrRegion) ) on UserPrincipalName | where Country != Country2 | where abs(datetime_diff('minute', TimeGenerated, TimeGenerated2)) < 60
New Admin Role Assignments
AuditLogs | where OperationName == "Add member to role" | where TargetResources[0].modifiedProperties[0].newValue contains "Global Administrator" | project TimeGenerated, InitiatedBy, TargetResources | order by TimeGenerated desc
Key Vault Secret Access
AzureDiagnostics | where ResourceType == "VAULTS" | where OperationName == "SecretGet" | project TimeGenerated, identity_claim_oid_g, requestUri_s, resultType_s | where resultType_s == "Success"
Defender for Cloud Alerts
SecurityAlert | where AlertSeverity in ("High","Medium") | where TimeGenerated > ago(24h) | summarize AlertCount=count() by AlertName, AlertSeverity, ProviderName | order by AlertCount desc
Anomalous Data Exfiltration
StorageBlobLogs | where OperationName == "GetBlob" | summarize TotalBytes=sum(ResponseBodySize) by CallerIpAddress, bin(TimeGenerated, 1h) | where TotalBytes > 1073741824 // 1 GB | order by TotalBytes desc
KQL tip: Use where before summarize to filter rows early. Use project to select only needed columns. Use bin() for time bucketing in summarize.
