DynamoDB

NoSQL Key-Value + Document
Data Model
Simple primary keyPartition key only (hash)
Composite primary keyPartition key + Sort key
Item size limit400 KB max per item
Attribute typesS (string), N (number), B (binary), BOOL, NULL, L (list), M (map), SS, NS, BS
Read Consistency
Eventually consistentHalf the RCU cost; stale reads possible
Strongly consistentFull RCU cost; always latest data; ConsistentRead=true
Transactional read2x RCU; ACID across multiple items/tables
Capacity Modes
ProvisionedSet RCU/WCU; auto-scaling available; cheaper at steady load
On-demandPay per request; scales instantly; 2.5x more expensive
RCU formulaCeil(item/4KB) × reads/s; ÷2 for eventual consistency
WCU formulaCeil(item/1KB) × writes/s
Indexes
LSI (Local)Same partition key, different sort key; created at table creation only; shares table capacity
GSI (Global)Any partition/sort key; can add after creation; own capacity; eventual consistency only
Advanced Features
DynamoDB StreamsOrdered item-level changes; 24hr retention; triggers Lambda
TTLAuto-delete items based on epoch timestamp attribute; free; ~48hr delay
TransactionsTransactWriteItems / TransactGetItems; up to 100 items; 2x cost
DAXIn-memory cache; microsecond reads; write-through; no Streams support
Batch operationsBatchGetItem (up to 100) / BatchWriteItem (up to 25); unprocessed items returned
Global TablesMulti-region active-active; requires Streams; eventual consistency cross-region
Hot partition fix: add random prefix/suffix to partition key to spread load across shards.
DynamoDB DAX Streams GSI/LSI

SNS & Kinesis

Pub/Sub & Streaming
SNS — Simple Notification Service
ModelPub/Sub; one publisher → many subscribers (fan-out)
Message size256 KB max
SubscribersLambda, SQS, HTTP/S, Email, SMS, Mobile push, Firehose
Fan-out patternSNS topic → multiple SQS queues (decoupled + parallel)
FIFO TopicsOrdered delivery; only SQS FIFO subscribers; dedup support
Message filteringFilter policy (JSON) per subscription; route by attribute
Message attributesKey-value metadata; used in filter policies
RetentionNo persistence — fire and forget; use SQS for durability
Kinesis Data Streams
Shards1 MB/s write, 2 MB/s read per shard; scale by adding shards
Data retention24 hrs default; up to 7 days (365 days long-term)
Partition keyDetermines which shard; same key → same shard (ordering)
GetRecordsPull model; up to 2 MB/s per shard; 5 reads/s per shard
Enhanced Fan-OutPush 2 MB/s per consumer per shard; HTTP/2; lower latency
Shard iterator typesTRIM_HORIZON, LATEST, AT_SEQUENCE, AFTER_SEQUENCE, AT_TIMESTAMP
Kinesis vs SQSKinesis = ordered streaming analytics; SQS = decoupled task queues
Kinesis Data Firehose
PurposeManaged delivery to S3, Redshift, OpenSearch, Splunk, HTTP
Near real-timeBuffer by size (1–128 MB) or time (60–900s); not real-time
Lambda transformInvoke Lambda to transform records before delivery
SNS does NOT persist messages. Always add SQS subscriber for reliable delivery. SNS FIFO → SQS FIFO only.

ElastiCache & S3

Caching & Object Storage
ElastiCache — Redis vs Memcached
RedisPersistence (RDB/AOF); pub/sub; sorted sets; multi-AZ; replicas; geospatial
MemcachedMulti-threaded; no persistence; no replication; simpler; horizontal scale
Redis ClusterData sharding across nodes; up to 500 nodes
Use Redis whenNeed persistence, pub/sub, Lua scripting, complex data structures
Caching Patterns
Lazy loadingRead: check cache → miss → load DB → write cache; stale data possible
Write-throughWrite: update DB + cache simultaneously; always fresh; unused data cached
Cache-asideApp manages cache explicitly (same as lazy loading)
TTLSet expiry to prevent stale data; balance freshness vs performance
S3 Key Facts
Object size0 bytes – 5 TB; multipart for >100 MB
Strong consistencyRead-after-write consistent (since Dec 2020)
Event notifications→ Lambda (async), SQS, SNS, EventBridge
Presigned URLsTemp access using creator's credentials; max 7 days
S3 SelectSQL query on object content; reduces data transferred
VersioningEnable per bucket; MFA Delete; delete marker on versioned delete
CORSConfigure on bucket for cross-origin browser requests
EncryptionSSE-S3 (default), SSE-KMS (audit trail), SSE-C (customer key), Client-side
S3 → Lambda always async. Lambda may need to retry — ensure idempotent handlers.

IAM & STS

Domain 2 — Security 26%
Policy Types
Identity-basedAttached to user/role/group; what they can do

Exam Tips & Patterns

Key Decision Rules
Keyword → Service Mapping
"Decouple"→ SQS (queue between components)
"Fan-out"→ SNS topic → multiple SQS queues
"Real-time streaming"→ Kinesis Data Streams
"Ordered messages"→ SQS FIFO or Kinesis (same shard)
"Exactly once"→ SQS FIFO with deduplication
"Cache DB reads"→ ElastiCache (Redis for complex) or DAX (DynamoDB only)
"Secrets rotation"→ Secrets Manager (not Parameter Store)
"Distributed tracing"→ X-Ray
"Canary deployment Lambda"→ CodeDeploy with Lambda alias traffic shifting
Common Traps
LSI vs GSI timingLSI: create time only; GSI: anytime (but eventually consistent)
Beanstalk Blue/GreenNot native B/G — done via URL/CNAME swap between environments
API GW 504 vs 502504 = Lambda timeout (>29s); 502 = unhandled exception in Lambda
SQS → Lambda typeIt's SYNCHRONOUS (Lambda polls) even though SQS is async queue
S3 → Lambda typeASYNCHRONOUS invocation; S3 fires event, Lambda processes
DynamoDB cross-accountUse IAM role + AssumeRole; NOT DynamoDB resource policy
CodeDeploy appspec hooksBeforeInstall → AfterInstall → ApplicationStart → ValidateService
CloudFormation DELETE_FAILEDS3 bucket not empty or DeletionPolicy: Retain set on resource
Tuning Decisions
High Lambda costReduce memory + optimize code; check reserved concurrency bottlenecks
DynamoDB 400 errorsProvisionedThroughputExceededException → add capacity or use exponential backoff
Lambda throttling429 error → increase reserved concurrency or account limit
X-Ray missing tracesCheck sampling rules; verify daemon running; check IAM write permissions
Read the question carefully — "most cost-effective" and "most secure" often have different answers. Eliminate clearly wrong options first.

Cognito User Pool= Authentication (who you are, JWT tokens)
Cognito Identity Pool= Authorization (AWS credentials, what you can do)
Envelope encryptionData key encrypts data; CMK encrypts data key
Auto-rotate secretsUse Secrets Manager (not Parameter Store)
EC2/Lambda authAlways use IAM roles, never hardcode keys

Troubleshooting Quick Rules
API GW 502Lambda unhandled exception (bad response format)
API GW 504Lambda exceeded 29s integration timeout
CF DELETE_FAILEDS3 bucket not empty OR DeletionPolicy: Retain
DynamoDB hot partitionAdd random suffix/prefix to partition key
X-Ray annotationsIndexed — use for filter expressions
X-Ray metadataNOT indexed — for debug data only
Must-Know Combos
LSICreated at table creation; same partition key
GSIAdded anytime; any keys; eventual consistency only
SQS → LambdaSync invocation; Lambda polls; on failure → DLQ
S3 → LambdaAsync invocation; auto retries 2x; DLQ for failures
