Model Evaluation Metrics

Domain 3 — Modeling
Classification Metrics
Accuracy(TP+TN)/total — misleading for imbalanced datasets; avoid as sole metric
PrecisionTP/(TP+FP) — minimize false positives (spam detection, fraud flagging)
Recall/SensitivityTP/(TP+FN) — minimize false negatives (cancer screening, fraud detection)
F1 Score2*(P*R)/(P+R) — harmonic mean; balanced metric for imbalanced classes
AUC-ROCmodel discrimination; 0.5=random, 1.0=perfect; threshold-independent metric
PR-AUCbetter than ROC-AUC for highly imbalanced data; focuses on minority class performance
SpecificityTN/(TN+FP) — true negative rate; complements recall in medical diagnostics
Regression Metrics
MAEmean absolute error; robust to outliers; interpretable in same units as target
MSE / RMSEpenalizes large errors more; RMSE in same units; sensitive to outliers
MAPEmean absolute percentage error; scale-independent; fails when target near zero
R²proportion of variance explained; 1.0=perfect; can be negative for very bad models
Bias-Variance Tradeoff
High Biasunderfitting — model too simple; increase complexity, add features, reduce regularization
High Varianceoverfitting — too complex; regularize, add more data, dropout, early stopping
L1 (Lasso)zeros out features; sparse solution; built-in feature selection capability
L2 (Ridge)shrinks all coefficients; no elimination; handles multicollinearity
ElasticNetcombination of L1+L2; tune with alpha and l1_ratio hyperparameters
Cross-Validation
k-Foldk=5 or 10 typical; rotate test fold through all k partitions; average score
Stratified k-Foldpreserves class ratio in each fold; use for imbalanced classification tasks
Leave-One-Outk = n samples; high variance estimate; use only for very small datasets
Precision vs Recall tradeoff: Raising classification threshold → increases precision, lowers recall. Lowering threshold does the opposite. F1 balances both at a given threshold.

Deep Learning

Domain 3 — Modeling
Architectures
CNNConv layers (feature maps), pooling (max/avg), fully connected; images & spatial patterns
RNN / LSTMsequential data; LSTM gates (input/forget/output/cell); solves vanishing gradient of plain RNN
Transformerself-attention; positional encoding; BERT (encoder) / GPT (decoder) / T5 (encoder-decoder)
Attention (Q/K/V)multi-head attention; scaled dot-product; O(n²) memory; foundation of modern NLP
GANGenerator + Discriminator adversarial training; image synthesis; data augmentation
Autoencoderencoder-decoder; latent space; anomaly detection; unsupervised dimensionality reduction
Activation Functions
ReLUmax(0,x); fast computation; dying ReLU problem → use LeakyReLU or ELU instead
Sigmoid[0,1] output; binary classification final layer; vanishing gradient at extreme values
Tanh[-1,1]; zero-centered; better than sigmoid for hidden layers; still vanishes
Softmaxmulti-class output probabilities; outputs sum to 1; pair with categorical cross-entropy
Regularization Techniques
Dropoutrandomly zero activations during training; typical rate 0.2–0.5; OFF at inference time
Batch Normalizationnormalize per mini-batch; faster convergence; acts as implicit regularizer
L1 / L2 Weight Regadd penalty to loss function; L1 sparsity, L2 smooth shrinkage
Data Augmentationflip/rotate/crop/color jitter; expand effective training set; prevents overfitting
Optimizers
SGDstochastic gradient descent; momentum helps escape local minima; needs careful lr tuning
Adamadaptive lr; combines momentum + RMSProp; most popular default; lr=1e-3 typical start
RMSPropgood for RNNs; adaptive lr per parameter; divides by moving avg of squared gradients
Training Stability
Vanishing Gradientuse ReLU activations, batch norm, residual (skip) connections, LSTM gating
Exploding Gradientgradient clipping (by value or norm); lower learning rate; weight initialization
LR too highloss diverges / NaN values; reduce lr or use warmup → cosine decay schedule
Batch sizelarger → stable gradients; smaller → regularizing effect; constrained by GPU memory (VRAM)

Common Gotchas & Quick Reference

Exam Traps
SageMaker Gotchas
Pipe vs File mode: Pipe streams from S3 directly (no EBS, fast startup for large data). File mode downloads to EBS first — use only for small datasets or random-access patterns.
RecordIO vs CSV: Built-in algos prefer RecordIO-Protobuf for speed. XGBoost uses CSV or libsvm. Convert to RecordIO if performance/memory is a concern on large datasets.
Warm Start: IDENTICAL type — HP ranges must be exactly same (continue tuning). TRANSFER type — ranges/algorithm can differ (new search space exploration).
VPC checklist: SageMaker in VPC requires (1) S3 VPC Gateway Endpoint, (2) ECR VPC Interface Endpoint, (3) SageMaker API Interface Endpoint. Missing any causes job failure.
Algorithm Selection Guide
Tabular supervisedXGBoost first choice; Linear Learner for speed/interpretability
Time seriesDeepAR for probabilistic output; Forecast service for AutoML/managed TS
Text classificationBlazingText (fast/custom); Comprehend custom (no-code managed)
RecommendationsFactorization Machines (sparse explicit); Personalize (managed implicit)
Anomaly detectionRandom Cut Forest (streaming real-time); Isolation Forest (offline batch)
Topic modelingLDA or NTM in SageMaker; Comprehend topics for managed no-code
Image tasksRekognition (no-code); SageMaker Image Classification/OD (custom labels)
Key Formulas
F12 × (Precision × Recall) / (Precision + Recall)
PrecisionTP / (TP + FP) — positive predictive value
RecallTP / (TP + FN) — sensitivity / true positive rate
Accuracy(TP + TN) / (TP + TN + FP + FN)
z-score(x − μ) / σ — standardization formula
Min-Max(x − min) / (max − min) — normalization to [0,1]
RMSE√(Σ(y­i − ŷ­i)² / n) — root mean squared error
