Exam Strategy & Key Tips

9 High-Value Tips
Test-Taking Strategy
  • Read the last sentence first. SAP scenarios are long — identify what you're optimizing for (cost, availability, security, performance) before reading the setup.
  • "Least operational overhead" almost always means managed services + automation, not manual scripts, self-managed EC2, or custom tooling.
  • Multi-account isolation pattern: SCPs restrict permissions → RAM shares resources → TGW connects networks. Know which tool solves which layer.
  • Global application pattern: Route 53 latency routing + CloudFront + Aurora Global DB is the standard architecture for worldwide low-latency apps.
DR Decision Tree
  • Seconds RTO → Multi-Site Active/Active across regions (highest cost)
  • Minutes RTO → Warm Standby (scaled-down but running full stack)
  • Hours RTO → Pilot Light (DB replicating; compute must start cold)
  • Day+ RTO → Backup & Restore (S3 snapshots; lowest cost)
Service Selection Rules
  • Direct Connect mentioned? Always pair with VPN backup — DX circuits go down during maintenance windows.
  • Heterogeneous DB migration (different engines) → DMS + SCT. Homogeneous (same engine) → DMS alone.
  • More than 3 VPCs needing full mesh → Transit Gateway beats VPC Peering on operational simplicity every time.
Cost optimization layering: Spot for fault-tolerant batch/HPC workloads → Reserved Instances or Savings Plans for steady-state baseline → On-Demand only for unpredictable spikes.
Common Traps
  • SCPs don't apply to the management account — never place production workloads there
  • VPC Peering is non-transitive — A↔B and B↔C does NOT give A↔C connectivity
  • RDS Read Replicas are async — they are NOT a DR solution by themselves (no automatic failover)
  • EBS volumes are AZ-locked — cannot be shared across AZs or multiple EC2 instances simultaneously
  • Pilot Light still has cold-start for EC2 — Warm Standby eliminates compute cold start
  • Lambda@Edge runs at CloudFront edge PoPs, not in your VPC — no VPC resource access by default
