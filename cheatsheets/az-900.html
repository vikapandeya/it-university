Exam Strategy
AZ-900 is conceptual — know WHEN to use each service, not HOW to configure it. Focus on differences between similar services (e.g., VPN Gateway vs ExpressRoute, ACI vs AKS, Availability Zones vs Availability Sets).
Key Theme
Shared Responsibility Model — who manages what shifts by IaaS/PaaS/SaaS. Know it cold. Also memorize SLAs: zones=99.99%, sets=99.95%, single premium SSD=99.9%.

Common Traps to Avoid
⚠Azure AD ≠ on-prem AD DS. No Kerberos/LDAP in Azure AD.
⚠Availability Sets protect within one DC, NOT across DCs.
⚠GRS secondary is read-only UNLESS you initiate failover.
⚠Tags don't auto-inherit to child resources by default.
⚠Free/Basic tiers have NO SLA guarantee.
⚠Azure Policy evaluates existing resources (audit), not just new.
⚠RBAC is additive; Deny assignments always override Allow.
⚠Composite SLA = multiply (not average) individual SLAs.
⚠Load Balancer = L4 (TCP/UDP). App Gateway = L7 (HTTP/HTTPS).
⚠Pricing Calculator ≠ TCO Calculator. One is estimate, one is compare.
⚠Spot VMs CAN be evicted at any time — never for critical workloads.
