Key Tip
systemd vs SysV: Know both systemctl and legacy init. XK0-005 is systemd-first but SysV may appear.
File Systems
XFS cannot shrink — ext4 can. btrfs has snapshots. Know which mkfs tool matches each FS type.
LVM Order
Create order: pv → vg → lv. Remove order: lv → vg → pv. Never reverse this sequence.
Permissions

Domain 1
Create order: PV → VG → LV  |  Remove order: LV → VG → PV
Physical Volumes (PV)

Volume Groups (VG)
vgcreate vg0 /dev/sdb vgcreate vg_data /dev/sdb /dev/sdc vgs vgdisplay vgextend vg0 /dev/sdc vgreduce vg0 /dev/sdc vgremove vg0 vgrename vg0 vg_new
Logical Volumes (LV)
lvcreate -L 10G -n lv0 vg0 lvcreate -l 100%FREE -n lv_data vg0 lvcreate -l +50%FREE -n lv_data vg0 lvs lvdisplay lvrename vg0 lv0 lv_new lvremove /dev/vg0/lv0
Extend LV + Resize FS
lvextend -L +5G /dev/vg0/lv0 lvextend -l +100%FREE /dev/vg0/lv0 resize2fs /dev/vg0/lv0 xfs_growfs /mount/point lvextend -r -L +5G /dev/vg0/lv0
lvextend -r resizes the filesystem automatically. For ext4: resize2fs. For XFS: xfs_growfs (XFS cannot shrink).

Permissions & ACLs

Domain 2 — Security
chmod Numeric
chmod 755 filerwxr-xr-x
chmod 644 filerw-r--r--
chmod 600 filerw------- (private key)
chmod 700 dirrwx------ (private dir)
chmod 777 filerwxrwxrwx (avoid!)
chmod -R 755 /dirRecursive
chmod Symbolic
chmod u+x file chmod g-w file chmod o=r file chmod u+x,g-w,o=r file chmod a+x file chmod a-x file
Ownership
chown user:group file chown user file chown :group file chown -R user:group /dir chgrp group file chgrp -R group /dir
Special Bits
chmod u+s fileSUID — runs as owner
chmod 4755 fileSUID numeric (4xxx)
chmod g+s dirSGID — inherit group
chmod 2755 dirSGID numeric (2xxx)
chmod +t dirSticky — only owner deletes
chmod 1777 /tmpSticky numeric (1xxx)
SUID on files: runs as file owner. SGID on dirs: new files inherit dir group. Sticky on dirs: only file owner can delete.
umask
umask umask 022 umask 027
File base: 666Dir base: 777
umask 022 → file666-022=644
umask 022 → dir777-022=755
Attributes (chattr/lsattr)
lsattr file chattr +i file chattr -i file chattr +a logfile
+i (immutable)Cannot modify/delete/rename
+a (append-only)Only append allowed
ACLs
getfacl file setfacl -m u:alice:rwx file setfacl -m g:devs:rx file setfacl -m o::r file setfacl -x u:alice file setfacl -b file setfacl -d -m u:alice:rwx dir setfacl -R -m u:alice:rwx dir
-d sets default ACL (inherited by new files). -b removes all ACLs. A + in ls -l output indicates ACLs exist.

SELinux

Domain 2 — Security
Status & Modes
getenforce sestatus sestatus -v setenforce 0 setenforce 1
EnforcingPolicy enforced, denials logged
PermissiveDenials logged only (not enforced)
DisabledSELinux turned off (requires reboot)
Config File
/etc/selinux/config — permanent mode change requires reboot
SELINUX=enforcing SELINUX=permissive SELINUX=disabled SELINUXTYPE=targeted
File Contexts
ls -Z /var/www/html/ chcon -t httpd_sys_content_t /var/www/html/index.html chcon -R -t httpd_sys_content_t /var/www/html/ restorecon -v /var/www/html/index.html restorecon -Rv /var/www/html/ semanage fcontext -l semanage fcontext -a -t httpd_sys_content_t '/data/web(/.*)?' semanage fcontext -d '/data/web(/.*)?'
chcon = temporary context change. restorecon = restore to policy default. semanage fcontext + restorecon = permanent.
Process Contexts
ps -Z ps -eZ | grep httpd id -Z
Booleans
getsebool -a getsebool httpd_can_network_connect setsebool httpd_can_network_connect on setsebool -P httpd_can_network_connect on setsebool -P httpd_enable_homedirs on semanage boolean -l
-P makes the boolean change persistent across reboots.
Ports
semanage port -l semanage port -l | grep http semanage port -a -t http_port_t -p tcp 8080 semanage port -d -t http_port_t -p tcp 8080
Troubleshooting & Audit
cat /var/log/audit/audit.log ausearch -m avc ausearch -m avc -ts recent ausearch -m avc | audit2allow ausearch -m avc | audit2allow -M mypolicy semodule -i mypolicy.pp audit2why < /var/log/audit/audit.log sealert -a /var/log/audit/audit.log
Before using audit2allow to allow a denial, understand why it was denied. Never blindly allow all AVC denials.
SSH Tip
ed25519 keys are preferred over RSA. Always set PermitRootLogin no and PasswordAuthentication no in sshd_config.
Troubleshoot Flow
Boot issues: dmesg → journalctl -b. Service: systemctl status → journalctl -u. Network: ip addr → ss -tulnp → ping.
Scripting Tip
Always quote variables: "$VAR" not $VAR. Use [[ not [ for comparisons. set -e exits on error.
Containers Tip
Podman = Docker API compatible, no daemon, rootless. buildah builds images without Docker daemon. Both preferred in RHEL environments.
